# Shared Knowledge

这里就是本 Team 的 **K0 Team 共享底座**。

所有角色都可以读取这里的内容。它不属于某一个 Agent，而是整个 Team 的共同规则层，用来统一输入字段、方法路由、报告格式、质量门、经营指标和客户版表达标准。

交付或上传时，这个文件夹必须和 `roles/` 一起带走。

```text
agent-ready/Teams/外贸经营定位专家Team/
├── knowledge/
│   ├── shared/   ← K0 Team 共享底座，所有 Agent 共用
│   ├── roles/    ← K1 角色私有知识切片
│   ├── cases/    ← K2 案例记忆
│   └── raw_refs/ ← K3 原始材料索引 / 深读追溯
└── roles/        ← 8 个 Agent 角色
```

不要把 K0 放进某一个角色文件夹里；否则其他 Agent 读不到共同规则。也不要把所有 K1 私有知识都混进 K0；否则每个 Agent 会读到太多无关内容。

当前已导出 7 个可交付共享知识块：

| 文件 | 用途 |
|---|---|
| `01_能力问题与质量门.md` | 8 个能力问题、5 条红线、评分表 |
| `02_输入字段与完整度.md` | 最小 12 字段、经营数据字段、数据成熟度、脱敏规则 |
| `03_方法路由卡.md` | BP_M00-BP_M07 方法路由，含 BP_M00.5 数据诊断 |
| `04_报告输出模板.md` | 最终报告结构和输出边界 |
| `05_经营罗盘指标映射.md` | 指标维度、上线前基线、7/30/90 天复盘 |
| `06_客户版表达与去黑话规则.md` | 客户版优先、内部黑话翻译、表达自检 |
| `07_经营数据证据门.md` | 数据可信度、数据到决策矩阵、输出等级 |

源知识来自：

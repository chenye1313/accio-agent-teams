# 02 输入字段与完整度

版本：v0.2

用途：把客户口述、Excel、店铺资料、后台截图、询盘记录、成交记录和公开竞品资料，统一成经营定位 Team 可调用的输入包。

本文件的核心升级：经营定位不能只靠品牌、产品和能力描述。正式决策前，必须尽量接入真实经营数据；数据不足时，只能输出假设版，不能伪装成经营结论。

## 1. 输入包总结构

Team 标准输入包由 6 类资料组成。

| 模块 | 输出对象 | 主要使用角色 |
|---|---|---|
| 企业与产品资料 | `client_profile.yaml` | Intake / 全角色 |
| 店铺后台经营数据 | `store_performance_snapshot.yaml` | Intake / Lead / MetricQualityGate |
| 询盘与成交数据 | `inquiry_sales_segments.yaml` | BuyerJTBD / BusinessModel |
| SKU / 产品表现数据 | `sku_performance_map.yaml` | BusinessModel / Differentiation |
| 页面内容与素材数据 | `page_content_audit_input.yaml` | Expression / SupplyMapping |
| 公开市场与竞品数据 | `market_competitor_scan.yaml` | SupplyMapping / Differentiation |

## 2. 最小 12 字段

没有这些字段，不应该直接生成最终报告。

## 4. 数据成熟度等级

这是经营数据层的成熟度，不替代基础资料完整度。

| 等级 | 条件 | 允许输出 |
|---|---|---|
| D0 | 只有店铺链接、公司介绍或公开页面 | 公开假设版；禁止写成经营决策 |
| D1 | 有 30/90 天店铺后台指标截图或手工表 | 可做经营诊断初稿，核心结论需标注待验证 |
| D2 | 有询盘文本、买家国家、买家身份或报价状态 | 可校准目标买家和买家任务 |
| D3 | 有订单、产品、成交金额区间、毛利区间、复购数据 | 可支持生意模式和主推产品决策 |
| D4 | 有上线前基线和上线后 7/30/90 天复盘数据 | 可做经营复盘和下一轮优化 |

## 5. 最小可用数据包

正式客户交付前，优先让商家准备以下资料。资料不全时，也能跑，但必须降级输出。

| 包 | 最小内容 | 目的 |
|---|---|---|
| 店铺后台包 | 近 30/90 天曝光、点击、访客、询盘、订单、Top 产品、Top 关键词、国家分布 | 判断流量、转化和市场入口 |
| 询盘样本包 | 20 条最近询盘，保留国家、买家类型、用途、采购量、成交状态 | 校准真实买家和采购任务 |
| SKU 表现包 | Top 10 SKU 的曝光、点击、询盘、成交、毛利区间 | 判断主推产品和真实机会 |
| 页面素材包 | 首页首屏、产品详情页、主图视频、证书、案例、评价截图 | 判断页面是否表达对了 |
| 竞品参照包 | 3-5 个竞品店铺、核心关键词搜索结果、价格/MOQ/证据资产 | 找市场常规说法和错位空间 |

## 6. 脱敏规则

客户可上传脱敏数据。Agent 只需要保留经营判断字段，不需要保留隐私字段。

| 敏感项 | 处理方式 | 保留字段 |
|---|---|---|
| 客户姓名 / 联系人 | 删除或替换为 `buyer_001` | 买家类型、国家、行业 |
| 邮箱 / 电话 / WhatsApp | 删除 | 无需保留 |
| 公司名 | 可脱敏为地区 + 类型 | 国家、买家身份、采购用途 |
| 成交价 | 可改为金额区间 | 客单价区间、利润区间 |
| 毛利 | 可改为高/中/低或区间 | 利润判断 |
| 聊天记录 | 删除个人信息后保留需求、焦虑、采购量、成交状态 | 买家任务分析 |

禁止把未脱敏客户隐私写入最终报告、案例库或可外发知识库。

## 7. 基础资料完整度等级

| 等级 | 条件 | Agent 行为 |
|---|---|---|
| A | 最小 12 字段齐全，且有 3 类证据资产 | 可生成报告 |
| B | 最小字段缺 1-3 项 | 先追问，再生成假设版初稿 |
| C | 缺目标买家或差异化 | 只生成访谈问题 |
| D | 只有产品和公司介绍 | 禁止生成定位，先做 Intake |

## 8. Intake 追问模板

当字段不足时，Intake 必须优先追问：

1. 你最想做的买家是哪一类？为什么？
2. 这类买家现在采购时最怕什么？
3. 他们如果不找你，一般会用什么替代方案？
4. 你最强的能力是哪一个？有没有证据？
5. 你最不想接哪类订单？
6. 你希望 6-12 个月后业务发生什么变化？
7. 你现在最想让阿里店铺页面解决什么问题？

如果用户愿意补经营数据，优先追问：

1. 近 30/90 天曝光、点击、访客、询盘、订单分别是多少？
2. 询盘最多来自哪些产品、关键词和国家？
3. 最近 20 条询盘里，哪些成交了，哪些没成交，原因是什么？
4. 哪些产品带来询盘，哪些产品真正成交，哪些产品利润更好？
5. 现在页面上有哪些证书、案例、客户评价和交付过程证据？

## 9. 进入下一步条件

| 条件 | 下一角色 |
|---|---|
| 最小字段齐全，且至少 D1 数据成熟度 | 进入 BP_M00.5 数据诊断，再进入 BusinessModel |
| 最小字段齐全，但数据成熟度 D0 | 可进入假设版，但 Lead 必须标注 `hypothesis_only` |
| 缺目标买家 | Intake 继续追问 |
| 缺差异化 | Intake 继续追问 |
| 缺证据资产 | 可进入假设版，但必须标注待补 |
| 缺 4 项以上最小字段 | 禁止生成报告 |

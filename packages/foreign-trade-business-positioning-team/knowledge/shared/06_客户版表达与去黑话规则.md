# 06 客户版表达与去黑话规则

版本：v0.1  
适用范围：全 Team，尤其是 Lead、Differentiation、Expression。

## 1. 总原则

一个好的经营定位老师，先看清问题本质、抓住关键变量、把需求说具体，具备洞察力，且表达通俗易懂；不装、不端、不搞内部黑话。

最终交付默认先给客户版。内部版只作为后台证据、质检和返工依据，不作为商家第一阅读材料。

## 2. 两版分工

| 版本 | 受众 | 保留内容 | 禁止内容 |
|---|---|---|---|
| 客户版 | 商家、阿里小二、培训现场 | 结论、为什么、怎么改、补什么资料、7/30/90 天行动 | Agent 编号、证据等级代码、状态机、内部质检黑话 |
| 内部版 | 讲师、架构师、Agent 调试者 | 证据等级、角色输出、红线、假设披露、返工路由 | 直接替代客户版交付 |

客户版不是把内部版删短，而是重新翻译成商家能听懂、能执行的经营语言。

## 3. 客户版报告标准结构

1. 一句话结论：我们建议你先做成什么。
2. 为什么不能继续原来的卖法。
3. 最应该先服务哪类客户。
4. 这类客户真正怕什么、想要什么。
5. 企业现在最能打的 3 个能力。
6. 现在不能承诺的内容。
7. 阿里国际站 7 屏怎么改。
8. 7 天 / 30 天 / 90 天行动清单。
9. 需要客户补充的资料。
10. 内部质量判断：只用一句人话说明“可先用 / 待补证 / 暂不上线”。

## 4. 去黑话翻译表

| 内部词 | 客户版说法 |
|---|---|
| M00/M01/M02/M03 | 资料采集 / 生意模式 / 买家分析 / 能力匹配 |
| JTBD / Job | 买家真正想完成的事 |
| A/B/C/D 级证据 | 已验证 / 有依据但待补证 / 推断 / 暂不能承诺 |
| PASS_WITH_GAP | 可以先做内部方案，但上线前要补资料 |
| internal_delivery_ready | 可作为内部方案讨论，不等于可直接上线 |
| product_ready | 可对外上线执行 |
| BOM | 配件清单 |
| DDP | 到门含税报价 |
| SLA | 响应承诺 |
| hybrid / mind_word / trust_first | 表达路径：先让客户听懂，再用证据建立信任 |
| unsupported_claims / rejected_claim | 不能说的话 |

## 5. 一句话定位标准

一句话定位必须同时回答三件事：

1. 服务谁。
2. 帮他解决什么具体问题。
3. 凭什么相信你能做。

合格表达像人话，不像内部标签。

坏例子：

- 承包商专属，1㎡ 起订的图纸级户外定制工厂。
- Your Shop Drawing. Our 1㎡ Container.

问题：概念锋利但工程感重，`1㎡ Container` 语义别扭，普通商家不一定立刻理解。

更适合客户版的表达：

- 专做海外庭院项目的小批量户外空间定制伙伴。
- 1㎡ 起订，按图按尺寸定制，凉亭、户外厨房、家具可一单配齐。

更适合阿里英文首屏的表达：

- Custom Outdoor Living Sets for Contractors, Starting from 1 m².
- Pergola, outdoor kitchen and furniture, configured by project and shipped together.

## 6. 表达自检

交付前必须问 6 个问题：

1. 客户能不能 10 秒内知道“我该服务谁”？
2. 客户能不能知道“我页面第一屏该怎么改”？
3. 客户能不能知道“哪些话现在不能说”？
4. 客户能不能知道“7 天内先做哪 3 件事”？
5. 是否出现了 Agent 编号、证据等级代码、状态机黑话？
6. 英文表达是否符合海外买家的自然理解，而不是中文直译？

任何一个问题失败，Lead 必须退回 Differentiation 或 Expression 重写客户版。

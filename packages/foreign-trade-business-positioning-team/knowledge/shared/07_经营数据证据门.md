# 07 经营数据证据门

版本：v0.1

用途：判断经营定位 Team 的核心结论是否足够支撑真实经营决策。它不是报告装饰，而是 Lead、Intake、BusinessModel、BuyerJTBD 和 MetricQualityGate 必须共同遵守的前置质量门。

## 1. 基本原则

1. 自有经营数据优先于公开抓取。
2. 成交数据优先于询盘数据。
3. 询盘文本优先于店铺页面语义。
4. 后台导出优先于截图。
5. 客户口述只能作为假设，不能直接作为经营结论。
6. Agent 推断必须显式标注，不能包装成事实。

## 2. 数据来源可信度

| 等级 | 数据来源 | 可支撑什么 |
|---|---|---|
| S | 平台后台导出、订单导出、CRM 导出、可核验财务/交易记录 | 经营决策、模式判断、复盘验证 |
| A | 后台截图、订单截图、询盘截图、证书原件、可公开案例证据 | 关键结论和页面信任资产 |
| B | 脱敏询盘文本、业务员跟进记录、客户访谈、报价记录 | 买家任务、采购焦虑、流失原因 |
| C | 店铺公开页面、产品标题、公开竞品页面、公开搜索结果 | 市场参照、表达对照、假设生成 |
| D | 客户口述、公司介绍、销售话术 | 初步假设和追问线索 |
| E | Agent 推断、行业常识、模型联想 | 只能写在待验证区 |

## 3. 输出等级

| 等级 | 定义 | 允许写法 |
|---|---|---|
| `decision_ready` | 有足够 S/A/B 级证据支撑，可指导经营动作 | “建议优先做...” |
| `discussion_ready` | 有部分证据，但关键数据缺失，需要客户确认 | “当前更倾向...” |
| `hypothesis_only` | 主要来自公开资料、口述或推断 | “这是待验证假设...” |
| `blocked` | 缺少基础事实，继续输出会误导决策 | “先补资料，不生成结论” |

## 4. 数据到决策矩阵

| 经营决策 | 必要证据 | 缺失时处理 |
|---|---|---|
| 当前最应该做哪盘生意 | 订单类型、成交客户、SKU 成交、毛利或复购 | 降级为 `discussion_ready`，不得写成确定主模式 |
| 第一优先买家是谁 | 询盘文本、成交买家、国家分布、采购量、用途 | 降级为 `hypothesis_only`，必须要求回填询盘 |
| 哪个产品应该主推 | Top 询盘产品、Top 成交产品、毛利、复购 | 只能写“建议测试”，不得要求马上重排店铺 |
| 哪些能力能作为核心优势 | 认证、案例、交付记录、流程证据、客户评价 | 没证据的优势进入待补清单 |
| 页面首屏怎么改 | 当前页面截图、目标买家、核心差异化、信任资产 | 可出草案，但必须标注素材缺口 |
| 7/30/90 天看什么指标 | 上线前基线、当前流量/询盘/订单指标 | 没基线时只能列观察指标，不能写提升承诺 |
| 是否进入正式客户交付 | 数据成熟度、证据等级、客户版可读性、红线通过 | 任一红线失败则 `blocked` 或返工 |

## 5. 决策证据充分性输出模板

```yaml
decision_sufficiency_gate:
  decision:
  output_level: "decision_ready | discussion_ready | hypothesis_only | blocked"
  data_maturity_grade: "D0 | D1 | D2 | D3 | D4"
  evidence_used:
    store_backend:
    inquiry_data:
    sales_data:
    sku_data:
    page_assets:
    competitor_public_data:
    customer_interview:
  missing_evidence:
  allowed_output:
  forbidden_output:
  next_data_request:
```

## 6. 降级规则

| 条件 | 强制输出等级 |
|---|---|
| 只有公开店铺链接 | `hypothesis_only` |
| 有后台指标但无询盘/成交 | 最高 `discussion_ready` |
| 有询盘但无成交 | 买家判断可 `discussion_ready`，生意模式最高 `discussion_ready` |
| 有成交但无毛利 | 可判断主推产品，但利润判断必须标注缺口 |
| 有订单/询盘/后台三类数据 | 可进入 `decision_ready` |
| 缺目标买家、主产品或增长目标 | `blocked` |

## 7. 客户版翻译规则

内部写法：

```text
output_level: hypothesis_only
```

客户版写法：

```text
因为目前还缺少后台和询盘数据，以下结论先作为第一版判断。等补充真实询盘和成交记录后，我们再校准目标买家和主推产品。
```

内部写法：

```text
blocked
```

客户版写法：

```text
现在不建议直接出最终定位。先补这几项资料，否则容易把方向判断错。
```

## 8. 红线

1. 不得把 `hypothesis_only` 写成确定结论。
2. 不得用公开竞品资料替代客户自有成交证据。
3. 不得在客户版暴露内部等级代码。
4. 不得引用未脱敏客户隐私。
5. 不得在缺基线时承诺指标提升比例。

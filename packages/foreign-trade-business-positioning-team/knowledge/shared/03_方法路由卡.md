# 03 方法路由卡

版本：v0.2

用途：这是 Team 的共享方法路由。Lead 用它决定调用哪个角色；每个角色用它确认自己的输入、输出和红线。

本版新增 `BP_M00.5`。经营数据必须在 BusinessModel 和 BuyerJTBD 之前进入判断链路；没有真实数据时，报告必须降级为假设版。

## BP_M00：输入采集与完整度判断

| 字段 | 内容 |
|---|---|
| Owner | Intake |
| Agent 类型 | 助手协作型 |
| 触发 | 客户提供 Excel、口述、公司介绍、店铺链接、后台截图、询盘记录或产品目录 |
| 输入 | `brand_name`、`main_products`、`growth_goal`、`current_business_model`、`target_buyer_types`、`priority_buyer`、`product_differentiation`、`customization_services`、`certifications`、`delivery_cycle`、`current_positioning` |
| 输出 | `client_profile_yaml`、`missing_fields`、`follow_up_questions`、`data_maturity_grade`、`missing_data_pack` |
| 红线 | 缺目标买家或差异化时，禁止生成最终报告 |

## BP_M00.5：经营数据诊断与决策证据门

| 字段 | 内容 |
|---|---|
| Owner | Intake + Lead |
| Agent 类型 | 数据归一 + 证据门 |
| 触发 | BP_M00 完成后；客户提供后台、询盘、成交、SKU、页面或竞品资料；或 Team 准备进入经营判断前 |
| 输入 | `client_profile_yaml`、`store_performance_snapshot`、`inquiry_sales_segments`、`sku_performance_map`、`page_content_audit_input`、`market_competitor_scan` |
| 输出 | `data_diagnosis_summary`、`decision_sufficiency_gate`、`allowed_output_level`、`missing_evidence` |
| 规则 | 后台 / 询盘 / 成交数据优先于公开抓取；公开抓取只能作市场参照；数据不足时必须降级输出 |
| 红线 | 只有公开店铺链接时，不得输出 `decision_ready` |

## BP_M01：生意模式判断

| 字段 | 内容 |
|---|---|
| Owner | BusinessModel |
| Agent 类型 | 专家顾问型 |
| 触发 | 已获得产品、订单类型、服务能力、目标买家；客户混合描述工厂、品牌、服务和贸易角色 |
| 输入 | `current_business_model`、`main_order_type`、`target_buyer_types`、`customization_services`、`rd_capabilities`、`delivery_cycle`、`sku_performance_map`、`sales_data_summary` |
| 输出 | `primary_business_model`、`secondary_business_models`、`unsuitable_models`、`model_evidence` |
| 规则 | 主模式必须解释“客户为什么找你，而不是找普通同行”；必须说明不适合走哪些模式；必须优先参考订单、SKU、毛利和复购数据 |
| 红线 | 只有客户口述且无订单 / SKU / 询盘证据时，只能输出 `discussion_ready` 或 `hypothesis_only` |

## BP_M02：买家 JTBD 与成功指标

| 字段 | 内容 |
|---|---|
| Owner | BuyerJTBD |
| Agent 类型 | 专家顾问型 |
| 触发 | 目标买家已知但痛点空泛；产品卖点无法解释采购动机 |
| 输入 | `priority_buyer`、`buyer_stage`、`main_products`、`buyer_pain_points_known`、`buyer_success_metrics_known`、`inquiry_sales_segments` |
| 输出 | `core_job`、`struggling_moment`、`current_workaround`、`desired_outcomes`、`purchase_anxiety`、`buyer_segment_confidence` |
| 规则 | Job 不得写成产品名；B2B 必须区分执行者、决策者、受益者；至少输出 3 个成功指标；真实询盘优先于公开店铺语义 |
| 红线 | 没有询盘或成交数据时，买家画像必须标注为推断 |

## BP_M03：供给能力映射

| 字段 | 内容 |
|---|---|
| Owner | SupplyMapping |
| Agent 类型 | 专家顾问型 |
| 触发 | 已有买家 Job，需要判断企业凭什么赢；客户列出很多优势但没有证据优先级 |
| 输入 | `core_job`、`desired_outcomes`、`product_differentiation`、`customization_services`、`rd_capabilities`、`delivery_cycle`、`certifications`、`partnerships`、`capacity`、`market_competitor_scan` |
| 输出 | `job_capability_fit`、`unsupported_claims`、`capability_gaps`、`market_common_claims`、`differentiation_openings` |
| 红线 | 无证据优势必须标记为待补，不可当核心差异化；竞品公开抓取不得替代自有能力证据 |

## BP_M04：差异化定位与取舍

| 字段 | 内容 |
|---|---|
| Owner | Differentiation |
| Agent 类型 | 专家顾问型 |
| 触发 | M01-M03 已完成；需要生成一句话定位、三条优势和禁用话术 |
| 输入 | `primary_business_model`、`priority_buyer`、`core_job`、`job_capability_fit`、`competitor_context`、`decision_sufficiency_gate` |
| 输出 | `positioning_statement`、`differentiation_points`、`forbidden_claims`、`expression_path` |
| 规则 | 先 JTBD，后定位抢词；成熟品类用定位抢词，顾问式供应链优先信任前置；定位必须能落到首屏 |
| 红线 | `hypothesis_only` 不得写成确定定位；没有取舍则退回 |

## BP_M05：信任资产与 7 屏表达

| 字段 | 内容 |
|---|---|
| Owner | Expression |
| Agent 类型 | 助手协作型 |
| 触发 | 已有定位，需要转成阿里国际站页面和内容 brief |
| 输入 | `positioning_statement`、`differentiation_points`、`certifications`、`partnerships`、`delivery_cycle`、`current_slogan`、`content_assets`、`page_content_audit_input` |
| 输出 | `slogan_options`、`visual_hammer`、`trust_proofs`、`seven_screen_brief`、`cta`、`page_change_priority` |
| 规则 | 每屏必须降低一种成本：识别、理解、信任、决策或行动；信任资产必须来自输入证据；客户版不得出现内部黑话 |

## BP_M06：经营罗盘指标影响

| 字段 | 内容 |
|---|---|
| Owner | MetricQualityGate / Lead |
| Agent 类型 | 专家顾问型 |
| 触发 | 任意定位建议生成前后；页面上线前；上线 7/30/90 天复盘 |
| 输入 | `positioning_statement`、`differentiation_points`、`seven_screen_brief`、`store_performance_snapshot`、`baseline_metrics`、`post_launch_metrics` |
| 输出 | `metric_impacts`、`baseline_metrics`、`first_7_day_metric`、`first_30_day_metric`、`first_90_day_metric`、`review_layer` |
| 红线 | 定位动作必须绑定至少 2 个经营罗盘指标；没有基线时必须标注“无法判断改善幅度” |

## BP_M07：Lead 质量门

| 字段 | 内容 |
|---|---|
| Owner | Lead |
| Agent 类型 | 专家顾问型 |
| 触发 | 报告生成前后；输出出现泛化、无证据、无取舍倾向 |
| 输入 | `business_positioning_report`、`client_profile_yaml`、`evidence_sources`、`decision_sufficiency_gate` |
| 输出 | `pass_fail`、`score`、`risk_points`、`revision_requests`、`customer_version_path`、`internal_appendix_path` |
| 红线 | 没有买家失败；没有 Job 失败；没有证据失败；不能落地页面失败；不能取舍失败；客户版出现内部黑话失败 |

## 推荐调用顺序

```text
BP_M00 输入采集
  -> BP_M00.5 经营数据诊断与决策证据门
  -> BP_M01 生意模式判断
  -> BP_M02 买家 JTBD
  -> BP_M03 供给能力映射
  -> BP_M04 差异化定位
  -> BP_M05 7 屏表达
  -> BP_M06 经营罗盘指标影响
  -> BP_M07 Lead 质量门
```

## 返工规则

| 失败点 | 退回角色 |
|---|---|
| 字段缺失 | Intake |
| 数据成熟度不足但写成决策 | Lead / Intake |
| 模式不清 | BusinessModel |
| Job 不成立 | BuyerJTBD |
| 优势无证据 | SupplyMapping |
| 定位无取舍 | Differentiation |
| 页面不能落地 | Expression |
| 指标不绑定 | MetricQualityGate |
| 客户版有内部黑话 | Lead / Expression |

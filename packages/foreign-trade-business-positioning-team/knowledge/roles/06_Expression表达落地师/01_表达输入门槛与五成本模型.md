# 01 表达输入门槛与五成本模型

版本：v0.1  
适用角色：`06_Expression表达落地师`

## 1. 核心判断

Expression 不负责重新定位，只负责把已通过的定位落成页面表达。

它要降低五类成本：

```text
识别成本 -> 理解成本 -> 信任成本 -> 决策成本 -> 行动成本
```

## 2. 输入门槛

必须具备：

```yaml
required_inputs:
  - positioning_statement
  - differentiation_points
  - proof_assets
  - forbidden_claims
```

可选增强：

```yaml
optional_inputs:
  - current_slogan
  - preferred_style
  - content_assets
  - case_studies
  - certifications
  - partnerships
```

缺 `positioning_statement`，不能写页面。  
缺 `proof_assets`，只能输出待补版 brief，不能写强信任表达。

## 3. 五成本模型

| 成本 | 页面任务 | 常见模块 |
|---|---|---|
| 识别成本 | 一眼知道你是谁 | 首屏定位、语言钉子、核心图 |
| 理解成本 | 迅速理解你卖什么/怎么帮我 | 产品结构、方案结构、服务流程 |
| 信任成本 | 相信你能做到 | 认证、案例、合作品牌、交付过程 |
| 决策成本 | 知道为什么现在询盘 | 对比、场景、方案包、风险降低 |
| 行动成本 | 降低询盘门槛 | CTA、表单、诊断入口、资料下载 |

## 4. 表达红线

- 不改写 Differentiation 的定位逻辑。
- 不展示 SupplyMapping 禁止使用的 claims。
- 不夸大合作品牌、认证或案例。
- 不写只有美感、没有业务作用的屏幕。
- 每一屏必须说明服务哪类买家决策。

# 06 输出模板与交接格式

版本：v0.1  
适用角色：`02_BusinessModel生意模式分析师`

## 1. 输出原则

BusinessModel 的输出要给下游 `03_BuyerJTBD买家任务分析师` 使用，不是给客户看的最终报告。

所以输出必须：

1. 明确主模式。
2. 明确辅模式。
3. 明确不适合模式。
4. 写清每个判断的证据。
5. 指出下游 BuyerJTBD 应优先分析哪类买家。

## 3. 状态规则

| `status` | 何时使用 | 下游动作 |
|---|---|---|
| `pass` | 主模式明确，证据足够 | 进入 BuyerJTBD |
| `need_more_info` | 候选模式存在，但关键证据不足 | 退回 Intake 补问指定字段 |
| `blocked` | 缺目标买家、订单类型或能力证据 | 禁止进入后续定位 |

## 4. 最小通过样例：HOTBLIZZARD

```yaml
business_model_analysis:
  role_id: "02_BusinessModel生意模式分析师"
  input_profile_id: "case2_hotblizzard"
  status: "pass"

  primary_business_model:
    name: "品牌孵化供应链型"
    score: 8
    one_sentence_reason: "它服务的是正在测款和起步扩张的潮牌主，成交理由是从创意、定制、小单测试到柔性量产的供应链陪跑。"
    target_buyer_bound:
      priority_buyer: "正在测试新品的街头潮牌创始人 / 小型潮流服饰品牌主"
      buyer_stage:
        - "起步"
        - "测款"
        - "扩张前期"
    order_type_bound:
      - "小单测试"
      - "深度定制"
      - "量产补货"
    evidence:
      buyer_evidence: "priority_buyer 与 buyer_stage 明确指向起步/测款品牌主。"
      order_evidence: "main_order_type 包含小单测试、深度定制、量产补货。"
      capability_evidence: "rd_capabilities 与 customization_services 支撑设计、打样、辅料和柔性量产。"
      profit_or_repeat_logic: "小单测款成功后进入补货和量产，提升客户生命周期价值。"
      flywheel_logic: "测款反馈越多，产品策略越准，客户复购和量产概率越高。"

  secondary_business_models:
    - name: "小单测试供应链型"
      score: 8
      reason: "7 天打样和小单测试订单强匹配。"
      boundary: "只能服务愿意试错和验证市场的买家。"
    - name: "柔性定制型"
      score: 7
      reason: "支持面料、工艺、logo、标签、辅料定制。"
      boundary: "需防止低利润杂单。"

  unsuitable_models:
    - name: "纯工厂供货型"
      reason: "当前定位和目标买家不是低价产能采购。"
      missing_evidence:
        - "缺少大货规模采购为主的订单结构证明。"

  flywheel_check:
    pass: true
    loop_summary: "小单测试 -> 市场反馈 -> 产品策略优化 -> 量产补货 -> 孵化案例增强。"
    metric_impacts:
      first_30_day:
        - "有效询盘率"
        - "询盘转化效率"
      first_90_day:
        - "报价订单率"
        - "老客户复购比"

  handoff_to_buyer_jtbd:
    priority_buyer: "正在测试新品的街头潮牌创始人 / 小型潮流服饰品牌主"
    buyer_stage:
      - "起步"
      - "测款"
    core_assumption_to_validate: "该买家真正购买的不是服装本身，而是低风险完成新品验证并进入放量补货。"
    suggested_jtbd_questions:
      - "他们当前从设计概念到可售样衣最卡在哪一步？"
      - "他们如何判断一款新品值得放量？"
      - "他们最害怕库存风险、交期风险还是设计落地失败？"
```

## 5. 禁止输出

以下输出不合格：

```yaml
primary_business_model: "制造商"
```

原因：制造商是身份，不是经营模式。

```yaml
primary_business_model:
  - "OEM"
  - "ODM"
  - "柔性定制"
  - "品牌孵化"
```

原因：主模式不能多个并列。

```yaml
one_sentence_reason: "客户实力强，产品丰富。"
```

原因：没有绑定买家、订单和能力证据。

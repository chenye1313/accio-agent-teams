# 03 B2B 模式选型决策树

版本：v0.1  
适用角色：`02_BusinessModel生意模式分析师`

## 1. 使用原则

决策树只用于缩小候选模式，不替代证据评分。

正确顺序：

```text
决策树缩小候选
  -> 证据字段评分
  -> 案例校准
  -> 输出主模式 / 辅模式 / 不适合模式
```

## 2. 快速决策树

```yaml
decision_tree:
  q1_buyer_buys_project_outcome:
    question: "买家买的是完整项目/场景结果，而不只是单品吗？"
    yes: "项目解决方案型"
    evidence:
      - CAD/3D/安装/配置方案
      - 多产品组合
      - 项目案例或工程客户

  q2_buyer_is_startup_brand_or_testing_seller:
    question: "优先买家是否处于起步、测款、扩张前期？"
    yes_next: "q2a"
    no_next: "q3"

  q2a_needs_strategy_plus_supply_chain:
    question: "企业是否参与产品策略、趋势判断、设计共创或内容素材？"
    yes: "品牌孵化供应链型"
    no_next: "q2b"

  q2b_needs_low_moq_fast_sampling:
    question: "企业是否主要提供小单、快样、多批次柔性生产？"
    yes: "小单测试供应链型 / 柔性定制型"
    no: "OEM / ODM 型"

  q3_buyer_buys_stable_capacity:
    question: "买家主要关心稳定产能、价格、交期、认证吗？"
    yes: "工厂供货型"
    no_next: "q4"

  q4_buyer_needs_private_label_or_product_development:
    question: "买家是否要自有品牌开发、贴牌、改款或产品开发？"
    yes: "OEM / ODM 型"
    no_next: "q5"

  q5_buyer_is_channel_partner:
    question: "买家是否是线下零售商、分销商、区域代理或批发渠道？"
    yes: "渠道分销型"
    no_next: "q6"

  q6_value_from_cross_category_sourcing:
    question: "价值是否来自跨品类组货、采购整合、质检和交付协调？"
    yes: "组货贸易型"
    no_next: "q7"

  q7_revenue_from_service_network_or_recurring:
    question: "收入是否来自订阅、抽成、服务费、数据资产或网络效应？"
    yes: "平台 / 服务型生意"
    no: "退回 Intake 补问订单类型和成交理由"
```

## 3. 常见组合

| 主模式 | 可搭配辅模式 | 不能混用的主线 |
|---|---|---|
| 项目解决方案型 | 柔性定制、渠道分销 | 纯工厂供货 |
| 品牌孵化供应链型 | 小单测试、柔性定制、ODM | 纯工厂供货 |
| 工厂供货型 | OEM、渠道分销 | 品牌孵化、平台型 |
| OEM / ODM 型 | 柔性定制、小单测试 | 项目解决方案，除非有项目交付证据 |
| 渠道分销型 | 工厂供货、组货贸易 | 品牌孵化，除非有品牌共创能力 |
| 组货贸易型 | 渠道分销、项目采购 | 平台型，除非有网络/数据机制 |

## 4. 不适合模式的判定

BusinessModel 必须输出不适合模式。判断方式：

```text
如果一个模式听起来高级，但没有对应买家、订单、能力、利润路径，就列入 unsuitable_models。
```

常见例子：

- 工厂想说自己是“平台”，但没有网络效应和服务收入。
- 代工厂想说自己是“品牌孵化”，但没有产品策略和内容共创能力。
- 贸易商想说自己是“解决方案”，但没有项目设计、集成、售后能力。
- 定制工厂想同时服务所有买家，导致主模式不清。

## 5. 追问模板

当模式不清楚时，退回 Intake 补问：

1. 你们最近 10 个订单分别来自哪类客户？
2. 每类客户买的是标准品、大货、试单、定制，还是项目方案？
3. 哪类订单毛利更好、复购更高？
4. 客户选择你们时，最看重价格、速度、定制、设计、认证、服务还是整体方案？
5. 哪些订单你们不想再接？为什么？

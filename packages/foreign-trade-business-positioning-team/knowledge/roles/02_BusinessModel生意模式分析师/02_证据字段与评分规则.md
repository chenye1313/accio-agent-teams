# 02 证据字段与评分规则

版本：v0.1  
适用角色：`02_BusinessModel生意模式分析师`

## 1. 评分目的

本文件用于避免 Agent 只根据客户口号下结论。

客户说“我们是品牌孵化供应链”，只能作为候选假设；必须用字段证据验证。

## 2. 阻断字段

任一阻断字段缺失，不能输出主模式，只能输出补问：

```yaml
blocking_fields:
  - target_buyer_types
  - main_order_type
  - main_products
  - customization_services_or_delivery_capability
```

其中 `customization_services_or_delivery_capability` 可以由以下任一字段支撑：

- `customization_services`
- `rd_capabilities`
- `delivery_cycle`
- `capacity`
- `certifications`
- `partnerships`
- `case_studies`

## 3. 五项评分

每个候选模式按 0-2 分评分，总分 10 分。

| 评分项 | 0 分 | 1 分 | 2 分 |
|---|---|---|---|
| 买家匹配 | 买家不清楚 | 买家泛化 | 买家明确且有优先级 |
| 订单匹配 | 订单类型缺失 | 订单类型存在但不稳定 | 订单类型与模式强一致 |
| 能力匹配 | 只有口号 | 有能力描述 | 有能力 + 数据/案例/流程证据 |
| 利润/复购 | 只解释成交 | 能解释利润或复购之一 | 能解释利润、复购或放量路径 |
| 飞轮潜力 | 一次性生意 | 有弱循环 | 有可持续正反馈循环 |

## 4. 分数解释

| 总分 | 判断 |
|---|---|
| 8-10 | 可以作为主模式候选 |
| 5-7 | 可作为辅模式或待验证模式 |
| 3-4 | 暂不建议主打 |
| 0-2 | 不适合模式 |

主模式除了分数最高，还必须满足：

1. 不触发阻断字段。
2. 能解释优先买家。
3. 能解释订单类型。
4. 能解释为什么不是普通同行。

## 5. 模式证据字段表

| 模式 | 必须看 | 强证据 | 弱证据 |
|---|---|---|---|
| 工厂供货 | `capacity`、`delivery_cycle`、`certifications` | 年产量、交期、认证、稳定大货 | “我们是源头工厂” |
| OEM / ODM | `rd_capabilities`、`customization_services` | 研发、打样、设计、贴牌、包装、版型 | 只写支持 logo |
| 柔性定制 | `main_order_type`、`delivery_cycle` | 小单、快样、多工艺组合 | “支持定制” |
| 小单测试供应链 | `buyer_stage`、`main_order_type` | 测款、首批、小单到量产路径 | MOQ 低 |
| 品牌孵化供应链 | `growth_goal`、`rd_capabilities`、`product_differentiation` | 产品策略、趋势判断、内容共创、增长路线 | “帮助客户做品牌” |
| 项目解决方案 | `target_buyer_types`、`case_studies`、`customization_services` | CAD/3D/安装/本地服务/整体配置 | 多产品目录 |
| 渠道分销 | `target_buyer_types`、`content_assets` | 展厅、手册、热销推荐、区域服务 | 有零售客户 |
| 组货贸易 | `main_products`、`supplier_network` | 跨品类组合、质检、交付协调 | 产品很多 |
| 平台 / 服务型 | `revenue_model`、`network_assets` | 订阅、抽成、数据资产、网络效应 | 有网站或系统 |

## 6. 风险标记

| 风险 | 触发条件 | 处理 |
|---|---|---|
| 口号化风险 | 只有定位话术，没有订单和能力证据 | 退回 Intake 补字段 |
| 万金油风险 | 同时想做 4 个以上模式 | 要求选主模式 |
| 低利润风险 | 小单/定制很多但没有流程化 | 标记为经营风险 |
| 错配买家风险 | 模式服务的买家与优先买家不一致 | 调整主模式 |
| 规模错配风险 | 想做品牌孵化但只有代工能力 | 降级为 OEM / 柔性定制 |

## 7. 输出最低要求

```yaml
business_model_score:
  primary_candidate:
    model: ""
    score: 0
    evidence:
      buyer_fit: ""
      order_fit: ""
      capability_fit: ""
      profit_or_repeat_logic: ""
      flywheel_logic: ""
  secondary_candidates: []
  unsuitable_models: []
  missing_fields: []
  return_to_intake: false
```

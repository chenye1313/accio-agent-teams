# 01 经营模式分类与判别规则

版本：v0.1  
适用角色：`02_BusinessModel生意模式分析师`  
用途：把客户的“公司身份 / 产品能力 / 订单习惯”转成可执行的经营模式判断。

## 1. 核心判断

BusinessModel 不判断“你是制造商还是贸易商”，而判断：

```text
客户为什么找你下单，而不是找普通同行？
这盘生意靠什么成交、靠什么复购、靠什么放大？
```

经营模式必须同时解释四件事：

1. 买家是谁。
2. 买家下的是什么类型订单。
3. 企业凭什么交付这个订单。
4. 这个模式如何带来更健康的经营指标。

## 2. 必要输入

进入本 Agent 前，至少需要：

| 字段 | 用途 |
|---|---|
| `main_products` | 判断产品复杂度、类目边界 |
| `current_business_model` | 捕捉客户自我描述，但不能直接照抄 |
| `main_order_type` | 判断成交逻辑：大货、小单、项目、补货、定制 |
| `target_buyer_types` | 判断模式服务谁 |
| `customization_services` | 判断定制深度 |
| `rd_capabilities` | 判断 ODM / 品牌孵化 / 解决方案能力 |
| `delivery_cycle` | 判断柔性和响应能力 |
| `capacity` | 判断规模供货和放量能力 |

缺 `target_buyer_types` 或 `main_order_type` 时，不能定主模式，必须退回 Intake。

## 3. 九类经营模式

### 3.1 工厂供货型

**定义**：客户主要购买稳定产能、价格、质量和交期。  
**典型买家**：进口商、批发商、大型渠道商、品牌采购部门。  
**成立证据**：

- 有明确产能、产线、交期、认证。
- 订单以稳定批量、大货、补货为主。
- 买家关心价格、良率、交期、合规。

**不成立信号**：

- 客户主要买创意、设计、解决方案、趋势判断。
- 企业没有规模产能和稳定交付证据。

### 3.2 OEM / ODM 型

**定义**：客户要借企业的开发能力、打样能力或制造能力完成自有品牌产品。  
**典型买家**：品牌方、渠道自有品牌、电商卖家。  
**成立证据**：

- 有打样、研发、版型、结构、配方、图案、工艺等能力。
- 能支持贴牌、改款、私模、包装、标签。
- 订单不是单纯买现货，而是围绕客户品牌规格生产。

**边界**：只支持印 logo 不等于 ODM；只有客户给图生产，更偏 OEM。

### 3.3 柔性定制型

**定义**：用低 MOQ、快打样、多规格、多工艺组合来服务不确定需求。  
**典型买家**：小品牌、趋势型卖家、活动/项目客户。  
**成立证据**：

- 低 MOQ、快样、工艺组合、SKU 多样。
- 能在小单和放量之间切换。
- 能承接多批次、小批量、高变化订单。

**风险**：如果没有成本控制和流程标准化，柔性会变成低利润杂活。

### 3.4 小单测试供应链型

**定义**：帮助买家用小单验证市场，再决定是否扩大生产。  
**典型买家**：新品牌、跨境电商卖家、DTC 品牌、内容型创业者。  
**成立证据**：

- 订单类型包含小单测试、样衣验证、首批试销。
- 有从小单到量产的路径。
- 能帮助降低库存风险和上新周期风险。

**关键边界**：小单测试不是“接小单”，而是服务买家的市场验证流程。

### 3.5 品牌孵化供应链型

**定义**：企业不仅生产，还参与产品策略、趋势判断、设计共创、内容素材或增长建议。  
**典型买家**：起步品牌主、小型潮牌创始人、产品能力薄弱的卖家。  
**成立证据**：

- 有产品策略、爆款判断、设计共创、内容素材、市场反馈机制。
- 不是只卖货，而是参与“从 idea 到 product-market fit”。
- 买家成功指标包括上新速度、爆款命中率、库存风险降低。

**误用红线**：没有策略共创和市场反馈能力时，不要包装成品牌孵化。

### 3.6 项目解决方案型

**定义**：客户购买的是一个场景/项目结果，不只是单品。  
**典型买家**：酒店、工程承包商、地产、商业空间、政府/机构项目。  
**成立证据**：

- 有设计图、方案配置、安装、售后、本地服务等能力。
- 产品之间有组合关系。
- 成交理由是降本、减少协调、降低项目风险。

**案例锚点**：Sweet Home 的户外空间方案明显属于此类主线，见 `process/cases/case1_sweethome_extract.md` R6-R8。

### 3.7 渠道分销型

**定义**：企业服务线下零售、区域代理、批发渠道，让对方更好卖。  
**典型买家**：经销商、零售商、批发商、区域代理。  
**成立证据**：

- 有产品手册、展厅支持、热销产品推荐、库存/配送支持。
- 目标不是单个终端消费者，而是帮助渠道卖给下一层客户。

**边界**：有渠道客户不等于渠道分销模式；必须有渠道赋能动作。

### 3.8 组货贸易型

**定义**：价值来自跨品类组合、供应商整合、采购协调、质检和交付。  
**典型买家**：小批发商、项目采购、跨品类卖家。  
**成立证据**：

- 产品跨类目组合明显。
- 企业能降低买家的搜品、验厂、质检、协调成本。

**风险**：如果没有供应链控制力，容易变成低信任中介。

### 3.9 平台 / 服务型生意

**定义**：价值不只来自单次卖货，而来自服务、工具、数据、网络或规则位置。  
**典型买家**：生态伙伴、商家、开发者、长期订阅客户。  
**成立证据**：

- 有持续服务、订阅、交易抽成、数据资产、网络效应。
- 能形成非线性增长或“庄家”位置。

**当前适用性**：多数外贸中小企业暂不适合直接定义为平台型，只能作为长期升级方向。

## 4. 判别顺序

```text
买家类型
  -> 订单类型
  -> 供给能力
  -> 盈利/复购逻辑
  -> 飞轮可持续性
  -> 主模式 + 辅模式 + 不适合模式
```

## 5. 主模式与辅模式规则

1. 主模式只能有一个。
2. 辅模式最多两个。
3. 主模式解释“为什么买你”。
4. 辅模式解释“还有哪些成交助力”。
5. 不适合模式必须写出来，防止客户同时主打所有方向。

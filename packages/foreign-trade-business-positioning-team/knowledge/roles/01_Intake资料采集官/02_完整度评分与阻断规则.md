# 02 完整度评分与阻断规则

版本：v0.1

角色：`01_Intake资料采集官`

## 核心判断

Intake 的第一责任不是推进流程，而是避免垃圾输入进入后续专家判断。

如果资料不足，必须阻断。

## 最小 12 字段

| 序号 | 字段 | 缺失后果 |
|---:|---|---|
| 1 | `brand_name` | 无法生成基础档案 |
| 2 | `main_products` | 无法判断生意模式和买家 Job |
| 3 | `main_markets` | 无法判断市场语境 |
| 4 | `growth_goal` | 无法判断经营意图 |
| 5 | `current_business_model` | 无法进入 BusinessModel |
| 6 | `target_buyer_types` | 无法进入买家分析 |
| 7 | `priority_buyer` | 无法生成定位 |
| 8 | `product_differentiation` | 无法判断差异化 |
| 9 | `customization_services` | 无法判断供给能力 |
| 10 | `certifications` 或其他信任资产 | 无法判断证据资产 |
| 11 | `delivery_cycle` | 无法判断交付可信度 |
| 12 | `current_positioning` | 无法比较旧定位和新定位 |

## 完整度等级

### A 级：可进入完整报告

条件：

- 最小 12 字段齐全。
- 至少有 3 类证据资产。
- `priority_buyer` 明确。
- `product_differentiation` 不只是“质量好 / 价格低 / 服务好”。

行为：

- 输出 `client_profile.yaml`。
- 交给 BusinessModel。
- 标注可生成经营定位报告。

### B 级：可进入假设版初稿

条件：

- 最小字段缺 1-3 项。
- 不缺 `target_buyer_types`。
- 不缺 `priority_buyer`。
- 不缺 `product_differentiation`。

行为：

- 输出 `client_profile.yaml`。
- 输出 `missing_fields.md`。
- 允许后续生成“假设版初稿”。
- 所有不确定字段必须标注待补。

### C 级：只能输出追问

任一条件触发：

- 缺 `target_buyer_types`。
- 缺 `priority_buyer`。
- 缺 `product_differentiation`。
- 买家只写“所有客户 / 全球客户 / B 端客户”。
- 差异化只写“质量好、服务好、价格低”。

行为：

- 禁止进入 BusinessModel。
- 禁止生成经营定位。
- 输出 5-8 个追问。

### D 级：只做资料采集

条件：

- 只有公司介绍和产品名。
- 最小 12 字段缺 4 项以上。
- 没有目标买家。
- 没有任何证据资产。

行为：

- 只输出调研表。
- 不进入后续 Agent。

## 红线字段

下面三个字段一旦缺失，必须阻断：

```text
target_buyer_types
priority_buyer
product_differentiation
```

原因：

- 没有买家，后续 JTBD 会空转。
- 没有第一优先买家，定位没有取舍。
- 没有差异化，后续表达会变成空泛文案。

## 证据资产评分

| 证据类型 | 强度 | 说明 |
|---|---:|---|
| 认证 / 检测 / 专利 | 高 | 可直接作为信任资产 |
| 合作品牌 / 项目案例 | 高 | 需注意授权和脱敏 |
| 交付数据 / 产能 / 周期 | 中高 | 可支撑供应链能力 |
| 客户评价 / 复购 | 中高 | 可支撑信任和服务 |
| 工厂照片 / 设备 | 中 | 可支撑生产能力 |
| 客户口述优势 | 低 | 需要二次验证 |

## 缺失字段追问规则

| 缺失字段 | 优先追问 |
|---|---|
| `growth_goal` | 未来 6-12 个月最想提升什么？ |
| `current_business_model` | 现在主要靠什么赚钱？ |
| `priority_buyer` | 如果只能选一类买家做首屏对象，你选谁？ |
| `product_differentiation` | 你最强的一点是什么？请给证据 |
| `certifications` | 有哪些认证、合作品牌、案例、评价或交付证明？ |
| `delivery_cycle` | 打样、量产、发货分别要多久？ |
| `current_positioning` | 现在页面或销售最常怎么介绍你们？ |

## 输出格式

### `completeness_grade`

```yaml
completeness_grade:
  grade: "A|B|C|D"
  reason:
    - ""
  missing_required_fields:
    - ""
  blocking_fields:
    - ""
  can_continue_to_business_model: true
  report_mode: "full|hypothesis|questions_only|intake_only"
```

### `missing_fields.md`

```md
# Missing Fields

## 阻断字段

- ...

## 建议补充字段

- ...

## 可继续但需标注假设的字段

- ...
```

### `follow_up_questions.md`

```md
# Follow Up Questions

1. ...
2. ...
3. ...
```

## 禁止行为

- 禁止资料不足时硬生成报告。
- 禁止替客户选择第一优先买家而不标注假设。
- 禁止把“公司实力强”直接写成差异化。
- 禁止把案例 2 的品牌孵化模式套给所有客户。

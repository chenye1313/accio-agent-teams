# 03 现场调研表：可交付版

版本：v0.1

角色：`01_Intake资料采集官`

用途：现场低门槛采集商家资料。商家不需要一次填完整，但缺关键字段时只能输出追问清单。

## A. 企业基础

| 问题 | 对应字段 |
|---|---|
| 英文品牌名 / 店铺名是什么？ | `brand_name` |
| 公司中文名 / 英文名是什么？ | `company_name_cn` / `company_name_en` |
| 国际站店铺链接 / 店铺 ID 是什么？ | `website_url` / `shop_id` |
| 主营产品是什么？ | `main_products` |
| 主营市场 / 国家有哪些？ | `main_markets` |
| 成立时间、工厂规模、员工数、办公室情况？ | `establishment_year` / `company_size` |
| 当前销售额、出口额、产能？ | `current_revenue` / `capacity` |

## B. 老板目标

| 问题 | 对应字段 |
|---|---|
| 未来 6-12 个月最想提升什么？ | `growth_goal` |
| 现在处于起步、增长、转型还是突破瓶颈？ | `business_stage` |
| 当前最大卡点是什么？ | `boss_priority` / `profit_constraints` |
| 明确不想接哪类订单 / 买家？ | `non_goals` |

## C. 生意模式

| 问题 | 对应字段 |
|---|---|
| 现在主要是工厂供货、OEM/ODM、定制、小单测试、工程项目、品牌孵化还是组货贸易？ | `current_business_model` |
| 主力订单类型是什么？ | `main_order_type` |
| MOQ、交付周期、价格逻辑是什么？ | `moq_policy` / `delivery_cycle` / `pricing_logic` |

## D. 产品与供给能力

| 问题 | 对应字段 |
|---|---|
| 主打产品是什么？ | `main_products` |
| 还有哪些配套产品？ | `supporting_products` |
| 产品主要功能是什么？ | `product_functions` |
| 可定制哪些内容？尺寸、颜色、面料、工艺、Logo、包装等 | `customization_services` |
| 产品或服务最不一样的地方是什么？ | `product_differentiation` |
| 研发、设计、打样、生产、交付上最强的一点是什么？ | `rd_capabilities` / `production_process` |
| 有哪些认证、专利、检测、质保、售后政策？ | `certifications` / `warranty` / `after_sales` |
| 有哪些合作品牌、项目案例、客户好评？ | `partnerships` / `case_studies` / `customer_reviews` |

## E. 目标买家

| 问题 | 对应字段 |
|---|---|
| 最想服务哪几类买家？ | `target_buyer_types` |
| 如果只能选一类作为第一优先买家，你选谁？ | `priority_buyer` |
| 这类买家在哪些国家或区域？ | `buyer_region` |
| 这类买家处在什么阶段？起步、测款、扩张、稳定采购、项目采购？ | `buyer_stage` |
| 这类买家采购时最怕什么？ | `buyer_pain_points_known` |
| 他们如果不找你，一般会找谁或怎么解决？ | `current_workaround` |
| 他们怎样判断一次采购是成功的？ | `buyer_success_metrics_known` |

## F. 当前表达

| 问题 | 对应字段 |
|---|---|
| 现在的定位 / Slogan 是什么？ | `current_positioning` / `current_slogan` |
| 希望店铺首屏让客户一眼记住什么？ | `current_positioning` / `preferred_style` |
| 现有页面最缺什么？产品结构、信任资产、案例、CTA、视频、视觉风格？ | `page_modules_existing` / `content_assets` |

## G. 经营罗盘

| 问题 | 对应字段 |
|---|---|
| 当前最想改善哪个指标？点击率、询盘成本、有效询盘率、访客转询盘、询盘转报价、报价转订单、复购？ | `target_metrics` |
| 过去 30 天最明显的问题是什么？ | `recent_30_day_issue` |
| 过去 90 天最值得复盘的问题是什么？ | `recent_90_day_issue` |

## 现场判定

### 可以进入报告

满足：

- 最小 12 字段齐全。
- 有明确第一优先买家。
- 有差异化和至少 1 类证据资产。

### 只能追问

触发任一项：

- 没有第一优先买家。
- 没有差异化。
- 没有信任资产。
- 不知道当前增长目标。

### 只做采集

触发任一项：

- 只有产品名。
- 只有公司介绍。
- 最小 12 字段缺 4 项以上。

## 现场输出

Intake 现场输出不超过 4 件：

1. `client_profile.yaml`
2. `completeness_grade`
3. `missing_fields.md`
4. `follow_up_questions.md`

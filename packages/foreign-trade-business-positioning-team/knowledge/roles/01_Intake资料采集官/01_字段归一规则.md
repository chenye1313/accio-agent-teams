# 01 字段归一规则

版本：v0.1

角色：`01_Intake资料采集官`

来源：

- `knowledge-base/经营定位/02_输入字段_schema.md`
- `knowledge-base/经营定位/06_商家现场调研表.md`
- `knowledge-base/经营定位/07_语音访谈脚本.md`

## 角色边界

Intake 不做经营定位结论，不判断最终差异化，不写报告。

Intake 只做四件事：

1. 把客户输入转成 `client_profile.yaml`。
2. 检查最小 12 字段。
3. 判断资料完整度等级。
4. 输出缺失字段和追问清单。

## 输入来源

| 输入类型 | 处理方式 |
|---|---|
| Excel 表 | 按字段名 / 语义映射到 schema |
| 公司介绍 | 提取品牌、产品、市场、能力、证据资产 |
| 店铺链接 / 店铺文案 | 提取品牌、现有定位、主产品、信任资产 |
| 语音访谈转写 | 按问题段落归入字段 |
| 产品目录 | 提取主产品、配套产品、功能、差异化 |
| 阿里后台导出字段 | 提取市场、询盘、转化、经营指标线索 |

## 字段归一原则

### 1. 保留原话，先不美化

客户原话必须保留在 `raw_fields` 或 `evidence_sources` 中。

错误做法：

```yaml
product_differentiation:
  - "高端专业品牌供应链能力强"
```

正确做法：

```yaml
product_differentiation:
  - "客户原话：深度定制、多维度可定制、先测款再放量"
```

### 2. 判断和事实分开

Intake 可以整理事实，不要提前做判断。

| 类型 | 放在哪里 |
|---|---|
| 客户原始描述 | `raw_fields` |
| 可验证事实 | 对应结构字段 |
| Agent 推断 | `intake_notes.assumptions` |
| 待确认问题 | `missing_fields` / `follow_up_questions` |

### 3. 多值字段统一用 list

这些字段必须用 list：

- `main_products`
- `main_markets`
- `boss_priority`
- `current_business_model`
- `main_order_type`
- `target_buyer_types`
- `customization_services`
- `certifications`
- `partnerships`
- `buyer_pain_points_known`
- `buyer_success_metrics_known`

### 4. 不能确定时标注待确认

不要硬填。

```yaml
priority_buyer: "待确认：客户提到街头潮牌创业者和跨境服装卖家，需要选择第一优先买家"
```

### 5. 证据来源必须可追溯

每个关键字段最好有来源。

```yaml
evidence_sources:
  brand_name: "Excel R2 / 店铺首页 / 客户口述"
  main_products: "产品目录 / Excel 主打产品字段"
  delivery_cycle: "客户访谈 Q11 / Excel 交付周期字段"
```

## 字段映射表

### A. 企业基础

| 目标字段 | 可能来源 | 归一规则 |
|---|---|---|
| `company_name_cn` | 公司中文名、营业执照、表格 | 原样记录 |
| `company_name_en` | 英文公司名、店铺资料 | 原样记录 |
| `brand_name` | 品牌名、店铺名、Logo | 必填，若只有公司名则标注待确认 |
| `shop_id` | 店铺 ID、URL 子域名 | 提取短 ID |
| `website_url` | 店铺链接、官网链接 | 保留完整 URL |
| `main_markets` | 主营国家、平台市场、客户分布 | 转为国家 / 区域 list |

### B. 老板目标与经营意图

| 目标字段 | 可能来源 | 归一规则 |
|---|---|---|
| `growth_goal` | 6-12 个月目标、老板目标 | 必填；用业务结果表达 |
| `business_stage` | 起步、增长、转型、突破瓶颈 | 可多选，但需标主阶段 |
| `boss_priority` | 订单、利润、大客户、新市场、品牌感 | 转 list |
| `non_goals` | 不想接的订单、不想服务的客户 | 原样记录 |

### C. 生意模式

| 目标字段 | 可能来源 | 归一规则 |
|---|---|---|
| `current_business_model` | 工厂供货、OEM、ODM、定制、工程、品牌孵化 | 可多选，不做最终主模式判断 |
| `main_order_type` | 小单、批发、项目、补货、现货 | 转 list |
| `moq_policy` | MOQ、起订量 | 原样记录 |
| `pricing_logic` | 成本加成、动态报价、项目报价 | 不确定则待确认 |

### D. 产品与供给能力

| 目标字段 | 可能来源 | 归一规则 |
|---|---|---|
| `main_products` | 主打产品、类目、产品目录 | 必填，转 list |
| `product_differentiation` | 产品差异、特殊工艺、策略 | 必填，保留客户原话 |
| `customization_services` | 尺寸、颜色、面料、Logo、包装 | 转 list |
| `rd_capabilities` | 研发、设计、打样、数据分析 | 转 list |
| `capacity` | 年产量、月产能、销售额 | 原样记录 |
| `delivery_cycle` | 打样、量产、物流周期 | 必填，原样记录 |
| `certifications` | 认证、检测、专利 | 必填或用其他信任资产替代 |

### E. 目标买家

| 目标字段 | 可能来源 | 归一规则 |
|---|---|---|
| `target_buyer_types` | 零售商、批发商、品牌主、工程商 | 必填，转 list |
| `priority_buyer` | 最想服务的买家 | 必填，若多个候选则标待确认 |
| `buyer_stage` | 起步、测款、扩张、项目采购、稳定补货 | 转 list |
| `buyer_pain_points_known` | 客户痛点、采购焦虑 | 保留原话 |
| `buyer_success_metrics_known` | 成功指标、采购标准 | 保留原话 |

### F. 证据资产

| 目标字段 | 可能来源 | 归一规则 |
|---|---|---|
| `partnerships` | 合作品牌、项目客户 | 注意授权风险，先标原始来源 |
| `customer_reviews` | 好评、复购、评价截图 | 转 list |
| `case_studies` | 项目案例、客户案例 | 转 list |
| `factory_photos` | 工厂、产线、设备图片 | 记录有无，不上传图片时写描述 |
| `delivery_proof` | shipping、安装、交付过程 | 转 list |
| `social_proof` | 销量、覆盖国家、复购率 | 保留数字和来源 |

### G. 表达资产

| 目标字段 | 可能来源 | 归一规则 |
|---|---|---|
| `current_positioning` | 现有定位、店铺首屏、公司介绍 | 必填或标待补 |
| `current_slogan` | slogan、口号、广告语 | 原样记录 |
| `preferred_style` | 视觉风格、色系偏好 | 原样记录 |
| `content_assets` | 视频、图文、详情页、社媒素材 | 转 list |

## 输出要求

Intake 每次必须输出：

```text
client_profile.yaml
completeness_grade
missing_fields.md
follow_up_questions.md
intake_notes
```

其中 `client_profile.yaml` 必须包含：

```yaml
brand_name:
main_products:
main_markets:
growth_goal:
current_business_model:
target_buyer_types:
priority_buyer:
product_differentiation:
customization_services:
certifications:
delivery_cycle:
current_positioning:
evidence_sources:
raw_fields:
intake_notes:
```

# 05 输出模板与交接格式

版本：v0.1  
适用角色：`04_SupplyMapping供给能力映射师`

## 1. 输出原则

SupplyMapping 的输出交给 `05_Differentiation差异化定位师`。它不直接写定位语，只交付：

1. 哪些能力能支撑买家 Job。
2. 哪些优势有证据。
3. 哪些说法不能用。
4. 哪些能力缺口需要补资料。

## 2. 标准输出 YAML

```yaml
supply_mapping_analysis:
  role_id: "04_SupplyMapping供给能力映射师"
  input_profile_id: ""
  status: "pass" # pass | need_more_info | blocked

  job_capability_fit:
    - desired_outcome: ""
      buyer_role: ""
      required_capability: ""
      company_evidence:
        field: ""
        value: ""
        source: ""
      proof_strength: "A | B | C | D"
      fit_level: "strong | medium | weak | unsupported"
      can_be_core_advantage: true
      note: ""

  capability_evidence_table:
    - capability: ""
      evidence_type: "data | process | certification | case | partnership | service"
      evidence_value: ""
      source_field: ""
      proof_strength: "A | B | C | D"
      linked_desired_outcomes:
        - ""

  unsupported_claims:
    - claim: ""
      why_unsupported: ""
      needed_evidence:
        - ""
      follow_up_question: ""

  capability_gaps:
    - desired_outcome: ""
      missing_capability: ""
      implication: ""
      priority: "high | medium | low"

  handoff_to_differentiation:
    strongest_capabilities:
      - ""
    usable_proof_assets:
      - ""
    forbidden_claims:
      - ""
    differentiation_risk:
      - ""
```

## 3. 状态规则

| `status` | 条件 | 下游动作 |
|---|---|---|
| `pass` | 至少 2 条强/中等能力能支撑核心 Job | 进入 Differentiation |
| `need_more_info` | 有候选能力但证据不足 | 退回 Intake 补证据 |
| `blocked` | 核心 Job 没有任何能力支撑 | 禁止进入差异化 |

## 4. 最小通过样例：HOTBLIZZARD

```yaml
supply_mapping_analysis:
  role_id: "04_SupplyMapping供给能力映射师"
  input_profile_id: "case2_hotblizzard"
  status: "pass"

  job_capability_fit:
    - desired_outcome: "缩短从设计概念到可售样衣的周期"
      buyer_role: "job_executor"
      required_capability: "快速打样与设计研发"
      company_evidence:
        field: "delivery_cycle + rd_capabilities"
        value: "7 天打样；面料、图案、辅料/工艺研发；快速打样"
        source: "case2_hotblizzard_profile.yaml"
      proof_strength: "B"
      fit_level: "strong"
      can_be_core_advantage: true
      note: "可支撑小单测试和品牌孵化，但需补真实打样案例。"

    - desired_outcome: "降低首批备货压错款的概率"
      buyer_role: "decision_maker"
      required_capability: "小单测试到柔性量产"
      company_evidence:
        field: "main_order_type + current_business_model"
        value: "小单测试、深度定制、量产补货；柔性制造"
        source: "case2_hotblizzard_profile.yaml"
      proof_strength: "B"
      fit_level: "strong"
      can_be_core_advantage: true
      note: "需补从小单到量产的客户案例。"

  unsupported_claims:
    - claim: "爆款导向"
      why_unsupported: "目前缺少真实爆款案例或测款数据。"
      needed_evidence:
        - "客户测款到补货案例"
        - "销售反馈或平台数据"
      follow_up_question: "有没有某个客户从小单测试到稳定补货的完整案例？"

  handoff_to_differentiation:
    strongest_capabilities:
      - "小单测试到柔性量产的衔接能力"
      - "多维定制和快速打样能力"
      - "合规认证与大牌合作信任资产"
    usable_proof_assets:
      - "7 天打样 + 45 天量产"
      - "Oeko-Tex 100 / 16 CFR 1610 / CP65"
      - "美特斯邦威 / New Balance 合作"
    forbidden_claims:
      - "爆款制造专家"
      - "全链路品牌孵化专家"
    differentiation_risk:
      - "品牌孵化说法需要案例支撑，否则应降级表达。"
```

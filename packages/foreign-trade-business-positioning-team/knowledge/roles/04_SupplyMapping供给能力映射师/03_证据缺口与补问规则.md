# 03 证据缺口与补问规则

版本：v0.1  
适用角色：`04_SupplyMapping供给能力映射师`

## 1. 用途

当客户描述了优势，但没有证据时，SupplyMapping 必须把它降级为 `unsupported_claims`，并生成补问。

## 2. 缺口类型

| 缺口 | 典型表现 | 补问方向 |
|---|---|---|
| 数据缺口 | 有“快/稳/强”，无数字 | 快多少、稳多少、产能多少 |
| 案例缺口 | 有能力，无客户案例 | 哪个客户、哪个项目、结果如何 |
| 过程缺口 | 有结果，无过程 | 怎么做到、流程是什么 |
| 对比缺口 | 说比同行好，无对比 | 同行通常怎么做，你们不同在哪里 |
| 复购缺口 | 说客户认可，无复购/评价 | 复购率、评价、合作年限 |
| 合规缺口 | 面向欧美市场，无认证 | 哪些认证、适用市场、证书有效期 |

## 3. 补问模板

### 3.1 针对交期

```text
你们说交期快，具体快在哪里？打样几天、大货几天、同行通常几天？有没有最近订单证明？
```

### 3.2 针对定制

```text
你们支持哪些定制项？哪些是标准可选，哪些需要研发？最复杂的定制案例是什么？
```

### 3.3 针对研发

```text
研发能力体现在哪个环节：材料、结构、图案、工艺、设备、数据分析，还是产品策略？
```

### 3.4 针对案例

```text
能不能提供 1-3 个代表案例：客户类型、需求、交付内容、结果、可公开证据？
```

### 3.5 针对品牌合作

```text
合作品牌具体合作了什么：供货、联名、OEM、ODM、长期补货，还是一次性订单？
```

## 4. 输出格式

```yaml
unsupported_claims:
  - claim: ""
    why_unsupported: ""
    needed_evidence:
      - ""
    follow_up_question: ""

capability_gaps:
  - desired_outcome: ""
    missing_capability: ""
    implication: ""
    priority: "high | medium | low"
```

## 5. 优先级规则

| 优先级 | 条件 |
|---|---|
| high | 直接影响主模式或核心 Job |
| medium | 影响辅模式或表达可信度 |
| low | 只影响补充信任资产 |

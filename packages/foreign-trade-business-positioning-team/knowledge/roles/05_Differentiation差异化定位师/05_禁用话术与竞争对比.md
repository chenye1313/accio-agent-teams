# 05 禁用话术与竞争对比

版本：v0.1  
适用角色：`05_Differentiation差异化定位师`

## 1. 禁用话术

以下话术默认禁止，除非 SupplyMapping 给出 A 级证据：

| 禁用词 | 原因 |
|---|---|
| 行业领先 | 需要市场份额、排名、第三方证据 |
| 一站式 | 需要完整方案和交付闭环 |
| 高品质 | 太泛，必须具体化到认证、良率、质检 |
| 服务好 | 太泛，必须具体化到响应、售后、流程 |
| 性价比高 | 容易陷入低价竞争 |
| 全链路 | 范围过大，需要每段证据 |
| 品牌孵化专家 | 需要客户成长案例 |
| 爆款制造 | 需要爆款数据或测款闭环 |

## 2. 竞争对比模板

```yaml
competitor_contrast:
  buyer_current_options:
    - option: ""
      weakness: ""
  generic_supplier_claims:
    - ""
  our_differentiation:
    claim: ""
    evidence:
      - ""
  tradeoff:
    we_prioritize: ""
    we_do_not_prioritize: ""
```

## 3. 取舍表达

必须写出“不优先服务谁”：

```yaml
not_for:
  - buyer: ""
    reason: ""
```

示例：

```yaml
not_for:
  - buyer: "只找最低价现货的大批发商"
    reason: "该定位服务的是需要小单测试和产品共创的潮牌主。"
```

## 4. 反泛化检查

定位输出后逐条问：

1. 删除品牌名后，同行能不能直接套用？
2. 这个定位有没有明确买家？
3. 这个定位有没有明确 Job？
4. 这个定位有没有强证据支撑？
5. 这个定位有没有不服务对象？

任一失败，必须返工。

# 03 华与华信任前置规则

版本：v0.1  
适用角色：`05_Differentiation差异化定位师`

## 1. 何时使用

当业务属于以下情况时，优先使用华与华式信任前置：

- 顾问式供应链。
- B2B 长周期、重信任成交。
- 企业需要承担客户产品研发、市场咨询或方案共创责任。
- 定位必须能转成页面动作、销售动作、服务动作。

## 2. 核心判断

华与华路径不先问“我要打败谁”，而先问：

```text
我为顾客承担什么咨询/研发/选择成本？
我怎样让每个动作都积累战略资产？
```

## 3. B2B 顾问化定位

B2B 企业可以把自己定义成客户某个关键任务的顾问：

| 原身份 | 顾问化表达 |
|---|---|
| 服装工厂 | 街头潮牌新品测试与柔性量产顾问 |
| 户外产品供应商 | 商业户外空间方案顾问 |
| 原料供应商 | 客户产品研发顾问 |
| 组货贸易商 | 跨品类采购风险控制顾问 |

## 4. 信任前置三件事

```yaml
trust_first:
  who_we_advise: ""
  what_decision_we_help: ""
  what_proof_we_show_first: []
```

优先展示：

- 代表案例。
- 交付过程。
- 认证 / 合规。
- 大牌合作。
- 关键流程。
- 可视化方案。

## 5. 与 Expression 的边界

Differentiation 只决定表达路径：

```yaml
expression_path:
  path_type: "trust_first | mind_word | hybrid"
  first_screen_claim: ""
  proof_assets_to_show:
    - ""
```

具体 7 屏页面怎么排版，交给 `06_Expression表达落地师`。

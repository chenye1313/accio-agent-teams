# 01 定位输入门槛与取舍原则

版本：v0.1  
适用角色：`05_Differentiation差异化定位师`

## 1. 核心判断

Differentiation 不负责“写得好听”，而负责做经营取舍。

一句合格定位必须回答：

```text
我们优先服务谁？
他要完成什么 Job？
我们凭什么比普通同行更适合？
我们不优先服务谁？
```

## 2. 进入门槛

必须具备以下输入：

```yaml
required_inputs:
  - primary_business_model
  - priority_buyer
  - core_job
  - job_capability_fit
  - unsupported_claims
```

缺任一项，不能生成定位语。

## 3. 定位四件套

```yaml
positioning_core:
  for_who: ""
  job_to_be_done: ""
  winning_capability: ""
  tradeoff_boundary: ""
```

## 4. 取舍原则

| 取舍 | 正确做法 |
|---|---|
| 买家取舍 | 先服务一个优先买家，不同时服务所有客户 |
| Job 取舍 | 先解决一个核心任务，不把全部痛点都写进定位 |
| 能力取舍 | 只使用 SupplyMapping 通过的强能力 |
| 话术取舍 | 不用无证据的“领先、专业、优质、全链路” |
| 场景取舍 | 定位必须能落到阿里首屏，而不只停在报告里 |

## 5. 定位句式

推荐句式：

```text
为[优先买家]，在[关键情境/Job]中，提供[核心能力支撑的经营结果]的[经营模式]。
```

示例：

```text
为正在测款的街头潮牌主，提供从小单验证到柔性量产的新品供应链伙伴。
```

## 6. 禁止直接使用的输入

以下内容不能直接写成定位：

- 客户原始 slogan。
- `current_positioning` 原话。
- 没有证据的 `product_differentiation`。
- C/D 级能力证据。
- 同行通用卖点。

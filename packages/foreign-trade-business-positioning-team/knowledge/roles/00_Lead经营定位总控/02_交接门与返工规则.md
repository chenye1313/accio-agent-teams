# 02 交接门与返工规则

版本：v0.1  
适用角色：`00_Lead经营定位总控`

## 1. 交接门原则

每个角色的输出都必须通过交接门，不能因为报告要完整就跳过。

```text
没有通过交接门 = 不允许进入下一角色。
```

## 2. 交接检查表

| 阶段 | 必须有 | 失败则退回 |
|---|---|---|
| Intake | `client_profile_yaml`、完整度、缺失字段 | Intake |
| BusinessModel | 主模式、辅模式、不适合模式、证据 | BusinessModel / Intake |
| BuyerJTBD | A 类买家、核心 Job、成功指标、替代方案 | BuyerJTBD / Intake |
| SupplyMapping | 能力证据表、无证据优势、能力缺口 | SupplyMapping / Intake |
| Differentiation | 定位语、三条优势、禁用话术、取舍 | Differentiation |
| Expression | 7 屏 brief、信任状、CTA、素材缺口 | Expression |
| MetricQualityGate | 指标映射、红线、评分、返工建议 | MetricQualityGate |

## 3. 返工路由

```yaml
revision_router:
  missing_fields: "01_Intake资料采集官"
  unclear_business_model: "02_BusinessModel生意模式分析师"
  invalid_job: "03_BuyerJTBD买家任务分析师"
  unsupported_capability: "04_SupplyMapping供给能力映射师"
  weak_positioning: "05_Differentiation差异化定位师"
  weak_page_brief: "06_Expression表达落地师"
  missing_metrics: "07_MetricQualityGate经营罗盘质检官"
```

## 4. 红线

任一红线失败，Lead 必须拒绝最终交付：

1. 没有明确目标买家。
2. 没有买家 Job。
3. 没有能力证据。
4. 不能转成阿里国际站页面或询盘动作。
5. 没有经营取舍。
6. 没有经营罗盘指标影响。

## 5. 源证据锚点

- `agent-ready/Teams/外贸经营定位专家Team/handoff_contract.yaml`：各角色 block_if。

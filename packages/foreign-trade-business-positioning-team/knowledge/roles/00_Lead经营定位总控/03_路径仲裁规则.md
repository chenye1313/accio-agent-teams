# 03 路径仲裁规则

版本：v0.1  
适用角色：`00_Lead经营定位总控`

## 1. 仲裁对象

Lead 主要仲裁三类冲突：

1. 生意模式冲突。
2. 定位方法路径冲突。
3. 证据强度冲突。

## 2. 生意模式冲突

| 冲突 | 仲裁 |
|---|---|
| 客户想同时做工厂供货、品牌孵化、平台 | 只保留一个主模式，其余作为辅模式或长期方向 |
| 目标买家和模式不匹配 | 以优先买家和订单类型为准 |
| 主模式没有证据 | 降级为候选假设，退回 BusinessModel / Intake |

## 3. 定位路径冲突

```yaml
path_arbitration:
  use_mind_word_if:
    - "成熟品类"
    - "竞争同质化"
    - "有可占据的简单词"
    - "有强证据支撑"
  use_trust_first_if:
    - "B2B 顾问式供应链"
    - "长周期决策"
    - "买家需要方案/研发/陪跑"
    - "信任资产比口号更关键"
  use_hybrid_if:
    - "既需要明确心智词"
    - "又需要用证据资产降低信任成本"
```

## 4. 证据冲突

| 情况 | 处理 |
|---|---|
| 定位很好但证据弱 | 降级为假设，不能确定交付 |
| 证据强但表达不锋利 | 退回 Differentiation |
| 页面好看但没有证据 | 退回 Expression / SupplyMapping |
| 指标无法绑定 | 退回 MetricQualityGate / Differentiation |

# 01 总控调度状态机

版本：v0.1  
适用角色：`00_Lead经营定位总控`

## 1. 核心职责

Lead 不替代各专家 Agent，而是负责：

1. 判断是否进入下一阶段。
2. 检查各角色输出是否互相一致。
3. 决定返工路径。
4. 汇总最终报告。
5. 标注证据结论与待验证假设。

## 2. 标准调用顺序

```text
01_Intake
  -> 02_BusinessModel
  -> 03_BuyerJTBD
  -> 04_SupplyMapping
  -> 05_Differentiation
  -> 06_Expression
  -> 07_MetricQualityGate
  -> 00_Lead final synthesis
```

## 3. 状态机

```yaml
lead_state_machine:
  intake_ready:
    next: "business_model"
    block_if:
      - "missing required profile fields"
  business_model_ready:
    next: "buyer_jtbd"
    block_if:
      - "primary_business_model empty"
      - "model has no evidence"
  buyer_jtbd_ready:
    next: "supply_mapping"
    block_if:
      - "core_job contains only product name"
      - "desired_outcomes empty"
  supply_mapping_ready:
    next: "differentiation"
    block_if:
      - "all differentiators are unsupported"
  differentiation_ready:
    next: "expression"
    block_if:
      - "positioning_statement has no buyer"
      - "forbidden_claims empty"
  expression_ready:
    next: "metric_quality_gate"
    block_if:
      - "seven_screen_brief less than 7 screens"
      - "trust_proofs empty"
  metric_gate_ready:
    next: "final_report"
    block_if:
      - "metric_impact_table empty"
      - "redline failed"
```

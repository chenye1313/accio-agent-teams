# 02 B2B 多角色采购任务拆解

版本：v0.1  
适用角色：`03_BuyerJTBD买家任务分析师`

## 1. 核心判断

B2B 采购不是一个人在买。

BuyerJTBD 必须区分：

```text
谁发起需求、谁评估方案、谁拍板、谁使用、谁承担失败后果。
```

如果只写“目标客户是品牌主 / 酒店 / 批发商”，不合格。

## 2. 五类角色

| 角色 | 定义 | 需要识别的 Job |
|---|---|---|
| Job Executor | 真正执行任务的人 | 怎么把任务做成 |
| Buyer / Purchaser | 负责采购和比价的人 | 怎么买得稳、合规、可解释 |
| Decision Maker | 拍板或承担预算的人 | 怎么降低商业风险、提高回报 |
| Influencer | 影响选择的人 | 怎么证明该方案值得信任 |
| Beneficiary | 最终受益者或下游客户 | 使用体验、销售结果或项目结果 |

## 3. 外贸 B2B 常见采购委员会

### 3.1 项目解决方案型

| 角色 | 常见对象 | 核心关注 |
|---|---|---|
| Job Executor | 项目经理、设计师、承包商 | 方案能不能落地 |
| Buyer | 采购经理 | 价格、交期、合规、付款 |
| Decision Maker | 酒店老板、地产负责人 | 项目风险、总成本、效果 |
| Influencer | 设计顾问、本地安装商 | 方案专业性 |
| Beneficiary | 终端使用者、住客、业主 | 空间体验 |

### 3.2 品牌孵化供应链型

| 角色 | 常见对象 | 核心关注 |
|---|---|---|
| Job Executor | 创始人、产品负责人、设计师 | 设计概念能否变成可售产品 |
| Buyer | 运营/采购负责人 | MOQ、打样、成本、交期 |
| Decision Maker | 品牌主、老板 | 库存风险、爆款概率、现金流 |
| Influencer | 社媒运营、买手、渠道伙伴 | 市场趋势和内容素材 |
| Beneficiary | 品牌粉丝、终端消费者 | 风格、舒适度、社交认同 |

### 3.3 工厂供货 / OEM 型

| 角色 | 常见对象 | 核心关注 |
|---|---|---|
| Job Executor | 采购执行、产品开发 | 规格匹配、样品、沟通效率 |
| Buyer | 采购经理 | 价格、交期、认证、付款 |
| Decision Maker | 供应链负责人、品牌老板 | 稳定性、风险、利润 |
| Influencer | 质检、法务、销售团队 | 合规和可销售性 |
| Beneficiary | 下游渠道/消费者 | 产品表现和供货稳定 |

## 4. 角色冲突

同一个订单中，不同角色的成功标准可能冲突。

| 冲突 | 表现 | 处理 |
|---|---|---|
| 采购要低价，老板要低风险 | 价格谈判压低质量空间 | 输出双成功指标 |
| 设计师要效果，采购要交期 | 方案反复修改 | 标出决策优先级 |
| 品牌主要爆款，供应链要稳定 | 趋势速度和生产确定性冲突 | 标出试单到量产路径 |
| 渠道要好卖，工厂要标准化 | SKU 和库存复杂 | 标出热销组合和淘汰机制 |

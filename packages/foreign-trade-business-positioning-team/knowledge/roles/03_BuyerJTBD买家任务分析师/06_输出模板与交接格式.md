# 06 输出模板与交接格式

版本：v0.1  
适用角色：`03_BuyerJTBD买家任务分析师`

## 1. 输出原则

BuyerJTBD 的输出要交给 `04_SupplyMapping供给能力映射师`，用于判断企业能力是否能支撑买家任务。

输出必须：

1. 明确优先买家。
2. 区分采购角色。
3. 写出不含产品名的核心 Job。
4. 写出当前替代方案。
5. 至少输出 3 个 Desired Outcomes。
6. 写出雇用/解雇标准。

## 3. 状态规则

| `status` | 何时使用 | 下游动作 |
|---|---|---|
| `pass` | Job、角色、成功指标、替代方案清楚 | 进入 SupplyMapping |
| `need_more_info` | 有目标买家，但缺采购情境或成功指标 | 退回 Intake 补问 |
| `blocked` | 优先买家不清或 Job 含产品名 | 禁止进入后续差异化 |

## 4. 最小通过样例：HOTBLIZZARD

```yaml
buyer_jtbd_analysis:
  role_id: "03_BuyerJTBD买家任务分析师"
  input_profile_id: "case2_hotblizzard"
  status: "pass"

  priority_buyer:
    name: "正在测试新品的街头潮牌创始人 / 小型潮流服饰品牌主"
    stage:
      - "起步"
      - "测款"
      - "扩张前期"
    source_from_business_model: "品牌孵化供应链型"

  core_job:
    statement: "在趋势窗口有限、库存风险高的情况下，用小单和快速打样把设计概念验证成可售新品，并在反馈有效后进入稳定补货。"
    functional_dimension: "完成新品验证和补货决策"
    emotional_dimension: "降低压错库存和错过趋势的焦虑"
    social_dimension: "让品牌看起来更专业、更稳定、更有风格"
    forbidden_product_words_checked: true

  struggling_moment: "品牌主有设计概念或趋势机会，但无法确定是否值得放量生产。"

  current_workaround:
    - "找普通服装工厂直接打样"
    - "大量下单赌爆款"
    - "使用现货改标"

  desired_outcomes:
    - statement: "缩短从设计概念到可售样衣的周期"
      outcome_type: "time"
      buyer_role: "job_executor"
      linked_capability_needed:
        - "快速打样"
        - "图案与工艺研发"
    - statement: "降低首批备货压错款的概率"
      outcome_type: "risk"
      buyer_role: "decision_maker"
      linked_capability_needed:
        - "小单测试"
        - "数据分析"
    - statement: "提高小单测试反馈对量产决策的可信度"
      outcome_type: "trust"
      buyer_role: "decision_maker"
      linked_capability_needed:
        - "柔性量产"
        - "产品策略研发"

  handoff_to_supply_mapping:
    core_job: "低风险完成新品验证并进入稳定补货"
    desired_outcomes_to_match:
      - "缩短从设计概念到可售样衣的周期"
      - "降低首批备货压错款的概率"
      - "提高小单测试反馈对量产决策的可信度"
    capability_keywords_to_check:
      - "7 天打样"
      - "45 天量产"
      - "面料研发"
      - "图案设计研发"
      - "数据分析"
```

## 5. 禁止输出

```yaml
core_job: "买卫衣"
```

原因：产品名不是 Job。

```yaml
desired_outcomes:
  - "质量好"
  - "服务好"
```

原因：不是可判断的成功指标。

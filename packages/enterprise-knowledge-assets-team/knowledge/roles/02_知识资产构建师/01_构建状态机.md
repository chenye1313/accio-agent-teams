# 01 构建状态机

版本：v0.1  
适用角色：`02_知识资产构建师`

## 1. 标准流程

```text
接收上游资料包
  -> 检查来源 / 脱敏 / 权限
  -> 术语与实体标准化
  -> 资料覆盖拆分
  -> 企业信息抽取
  -> 产品信息抽取
  -> 客户问答抽取
  -> 客户案例抽取
  -> 组织实践标签沉淀
  -> 交接给知识质量与分发官
```

## 2. 状态机

```yaml
asset_building_state_machine:
  handoff_checked:
    next: "term_entity_normalization"
    block_if:
      - "source_index missing"
      - "permission_tags missing"
      - "unprocessed_sensitive_data"

  term_entity_normalized:
    next: "coverage_split"
    block_if:
      - "product/entity conflict unmarked"
      - "source language ambiguity unmarked"

  coverage_split_ready:
    next: "enterprise_product_extraction"
    block_if:
      - "library input plan missing"
      - "material type not mapped"

  enterprise_product_extracted:
    next: "qa_case_extraction"
    block_if:
      - "enterprise/product candidates missing source_id"
      - "product feature without use_case"

  qa_case_extracted:
    next: "practice_tagging"
    block_if:
      - "qa without buyer_context"
      - "case without problem_solution_result"

  practice_tagged:
    next: "handoff_to_quality"
    block_if:
      - "method externalized_as_fifth_library"
      - "inference_log missing"

  handoff_to_quality_ready:
    next: "03_知识质量与分发官"
    block_if:
      - "missing_fields not reported"
      - "source_id not retained"
```

## 3. 可带缺口通过

允许 `pass_with_gap`：

- 企业和产品资料完整，但客户问答不足。
- 有案例线索但缺结果。
- 有老板经验但缺证据。
- 后台数据仅作为证据层，未形成正式字段。

## 4. 必须阻断

- 无来源 ID。
- 未脱敏。
- 权限冲突。
- 用户要求跳过四库结构化直接生成内容。
- 资料中没有任何可抽取字段。

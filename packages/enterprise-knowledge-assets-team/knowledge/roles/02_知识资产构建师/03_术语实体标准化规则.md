# 03 术语实体标准化规则

版本：v0.1  
适用角色：`02_知识资产构建师`

## 1. 需要统一的实体

| 类型 | 示例 |
|---|---|
| 公司实体 | 公司名、品牌名、工厂名、店铺名 |
| 产品实体 | 产品线、SKU、型号、系列、材质 |
| 客户实体 | 买家类型、国家、行业、角色 |
| 认证实体 | ISO、CE、FDA、BSCI、专利、检测报告 |
| 渠道实体 | 阿里国际站、OKKI、官网、WhatsApp、邮件 |
| 证据实体 | 订单、询盘、后台截图、聊天记录、录音、证书 |

## 2. 标准输出

```yaml
term_entity_map:
  - raw_term: ""
    normalized_term: ""
    entity_type: "company/product/customer/certification/channel/evidence"
    source_id: ""
    confidence: "high/medium/low"
    notes: ""
```

## 3. 同义词处理

```yaml
synonym_map:
  - normalized_term: ""
    variants:
      - ""
    source_ids:
      - ""
```

## 4. 冲突处理

必须标记：

- 同一个产品多个名称。
- 同一个 SKU 多个型号写法。
- 同一个认证多个简称。
- 客户类型混用：buyer / distributor / wholesaler / brand owner。
- 行业词和场景词混用。

```yaml
conflict_terms:
  - raw_terms: []
    conflict_type: ""
    suggested_resolution: ""
    needs_lead_arbitration: true
```

## 5. 禁止

- 禁止自行创造产品名。
- 禁止删除原始称呼。
- 禁止把不确定实体写成确定统一名。
- 禁止把营销口号当成术语。

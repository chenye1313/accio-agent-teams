# 06 组织实践标签与方法内沉规则

版本：v0.1  
适用角色：`02_知识资产构建师`

## 1. 为什么要有组织实践标签

四库解决“资料能被调用”的问题；组织实践标签解决“企业做对事情的经验能不能复用”的问题。

方法经验不作为对外第五库，但必须在内部沉淀，否则企业知识库基座会退化成资料库。

## 2. 标签模型

```yaml
organization_practice_tag:
  scenario: ""
  process: ""
  role: ""
  judgment: ""
  action: ""
  standard: ""
  knowledge_used: ""
  mechanism: ""
  feedback: ""
  source_ids: []
  status: "verified/needs_validation/internal_only"
```

## 3. 来源

组织实践标签主要来自：

- 客户案例。
- 客户问答。
- 老板访谈。
- 销售复盘。
- 交付复盘。
- 丢单复盘。

## 4. 内沉位置

| 方法经验类型 | 内沉位置 |
|---|---|
| 成交方法 | 客户案例库 `reusable_method` |
| 异议处理方法 | 客户问答库 `recommended_usage` |
| 流程经验 | `organization_practice_tag.process` |
| 判断标准 | `organization_practice_tag.judgment` |
| 动作标准 | `organization_practice_tag.action` / `standard` |
| 复盘机制 | `organization_practice_tag.feedback` |

## 5. 验证状态

```yaml
practice_status:
  verified: "有案例或多次问答验证"
  needs_validation: "只有口述或单次经验"
  internal_only: "适合内部训练，不适合对外表达"
```

## 6. 禁止

- 禁止把方法经验包装成对外第五库。
- 禁止把单次经验写成普遍规律。
- 禁止没有来源的组织实践标签。

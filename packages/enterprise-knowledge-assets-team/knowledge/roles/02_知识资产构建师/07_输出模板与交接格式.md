# 07 输出模板与交接格式

版本：v0.1  
适用角色：`02_知识资产构建师`

## 1. 标准输出

```yaml
asset_builder_output:
  role_id: "02_知识资产构建师"
  status: "build_ready | pass_with_gap | blocked"

  term_entity_map: []
  synonym_map: []
  conflict_terms: []

  coverage_split_map:
    - asset_id: ""
      target_library: []
      extractable_fields: []
      missing_fields: []

  enterprise_info_candidates: []
  product_info_candidates: []
  customer_qa_candidates: []
  customer_case_candidates: []

  organization_practice_tags: []

  missing_fields:
    - target_library: ""
      field: ""
      reason: ""
      suggested_fix: ""

  inference_log:
    - item_id: ""
      inferred_field: ""
      basis: ""
      status: "needs_validation"

  handoff_to_quality_distributor:
    allowed: true
    package_items: []
    warnings: []

  user_summary: ""
```

## 2. 交接给知识质量与分发官

必须包含：

- 四库候选条目。
- 术语实体表。
- 组织实践标签。
- 缺失字段清单。
- 推断记录。
- 来源 ID。
- 权限状态。

## 3. 老板版摘要

```text
这批资料已经被拆成四类知识候选：
企业信息：{enterprise_count}
产品信息：{product_count}
客户问答：{qa_count}
客户案例：{case_count}

其中有些内容还只是经验或推断，不能直接当结论。
下一步要交给知识质量与分发官，检查来源、证据、隐私和能否分发给六组。
```

## 4. 失败输出

```yaml
blocked_output:
  status: "blocked"
  blocked_reason: ""
  return_to_role: "01_资料接收与托管师 or 00_Lead知识资产总控"
  required_fix: []
  allowed_partial_output: ""
```

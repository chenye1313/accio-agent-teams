# 01 质量分发状态机

版本：v0.1  
适用角色：`03_知识质量与分发官`

## 1. 总流程

```text
接收候选条目
  -> 检查交接完整度
  -> 检查来源与证据等级
  -> 检查隐私和公开权限
  -> 检查四库字段完整度
  -> 扫描 AI-Slop
  -> 生成质量判定
  -> 打包分发给六组
  -> 记录回流和更新请求
```

## 2. 状态定义

| 状态 | 含义 | 下一步 |
|---|---|---|
| `received` | 收到构建师交付包 | 进入交接完整度检查 |
| `handoff_incomplete` | 缺少来源、权限、字段或推断记录 | 退回 `02_知识资产构建师` 或 `01_资料接收与托管师` |
| `under_evidence_review` | 正在检查来源和证据等级 | 进入权限检查 |
| `under_privacy_review` | 正在检查脱敏和公开权限 | 进入字段验收 |
| `under_library_review` | 正在检查四库字段 | 进入 AI-Slop 扫描 |
| `quality_approved` | 通过质量门 | 进入分发打包 |
| `quality_approved_with_gap` | 可内部使用但有缺口 | 标注限制后分发或返工 |
| `needs_validation` | 有来源但证据不足 | 只进入待验证清单 |
| `internal_only` | 可内部使用但不可公开 | 仅分发给内部任务 |
| `rejected` | 不可使用 | 输出拒绝原因 |
| `distributed` | 已按六组任务分发 | 等待回流 |
| `feedback_received` | 收到六组反馈 | 生成更新请求 |

## 3. 交接完整度检查

构建师交付包必须包含：

```yaml
handoff_required_fields:
  - item_id
  - target_library
  - source_id
  - source_type
  - permission_status
  - candidate_content
  - missing_fields
  - inference_log
```

缺少 `source_id`、`permission_status` 或 `target_library`，不得进入质量评审。

## 4. 返工路由

| 问题 | 退回角色 | 要求 |
|---|---|---|
| 来源缺失 | `01_资料接收与托管师` | 补来源索引或标记为人工补充 |
| 权限未知 | `01_资料接收与托管师` | 补权限状态或降为内部不可外发 |
| 字段缺失 | `02_知识资产构建师` | 补四库字段或写明缺失原因 |
| 术语混乱 | `02_知识资产构建师` | 回到术语实体标准化 |
| 判断无依据 | `02_知识资产构建师` | 改为推断记录或待验证经验 |
| 分发边界冲突 | `00_Lead知识资产总控` | 重新定义调用边界 |

## 5. 完成条件

必须同时满足：

- 每条知识有来源。
- 每条知识有证据等级。
- 每条知识有权限状态。
- 每条知识有四库归属。
- 每条知识有质量判定。
- 每个分发包有使用边界。
- 每个分发包有回流要求。

# 02 证据等级与来源一致性规则

版本：v0.1  
适用角色：`03_知识质量与分发官`

## 1. 来源一致性检查

每条候选知识必须能回答：

```yaml
source_check:
  source_id: ""
  source_type: "document | interview | recording | chat | backend | manual"
  source_owner: ""
  received_at: ""
  permission_status: "public | anonymized | internal_only | unknown | restricted"
  source_excerpt_or_pointer: ""
  claim_supported: true
```

如果 `candidate_content` 的结论强于来源，只能降级为推断。

## 2. 证据等级

| 等级 | 可接受来源 | 可用边界 |
|---|---|---|
| A | 后台数据、订单、合同、成交记录、客户反馈、认证证书 | 可进入正式库，可用于确定结论 |
| B | PPT/PDF/官网/产品目录、聊天记录、录音转写、脱敏案例、交付记录 | 可进入正式库，但建议保留来源说明 |
| C | 老板访谈、销售口述、团队复盘、人工补充且有业务合理性 | 只能作为待验证经验或内部标签 |
| D | 无来源、泛泛口号、模型臆测、无法对应资料的判断 | 拒绝进入正式库 |

## 3. 跨来源冲突处理

当多个来源冲突时：

```yaml
conflict_resolution:
  same_fact_conflict:
    priority: "A > B > C > D"
  time_conflict:
    priority: "newer_source_first"
  permission_conflict:
    priority: "more_restrictive_permission_first"
  oral_vs_document_conflict:
    action: "retain_both_and_mark_needs_confirmation"
```

不得为了让知识更好看而删除冲突。

## 4. 隐私与权限复核

必须脱敏：

- 客户姓名。
- 联系方式。
- 邮箱、电话、WhatsApp。
- 未授权公司名。
- 精确成交价。
- 合同条款、付款账期。
- 员工个人隐私。

权限规则：

```yaml
permission_gate:
  public:
    can_use_for: ["public_content", "sales_material", "internal_decision"]
  anonymized:
    can_use_for: ["public_content_after_anonymization", "sales_material", "internal_decision"]
  internal_only:
    can_use_for: ["internal_decision", "sales_training"]
  unknown:
    can_use_for: ["quality_review_only"]
  restricted:
    can_use_for: ["do_not_distribute"]
```

## 5. 质量判定

evidence_source_quality_result:

# 06 输出模板与交接格式

版本：v0.1  
适用角色：`03_知识质量与分发官`

## 1. 标准输出

```yaml
quality_distribution_output:
  role_id: "03_知识质量与分发官"
  status: "approved | approved_with_gap | blocked"

  evidence_review:
    approved_items: []
    approved_with_boundary_items: []
    needs_validation_items: []
    internal_only_items: []
    rejected_items: []

  ai_slop_findings:
    - item_id: ""
      issue_type: ""
      evidence: ""
      required_fix: ""

  quality_scores:
    - item_id: ""
      accuracy: 0
      completeness: 0
      traceability: 0
      usability: 0
      total: 0
      verdict: ""

  six_group_distribution_plan:
    packages: []

  feedback_loop_requirements:
    - target_group: ""
      required_feedback: []

  update_requests: []
  handoff_to_lead:
    allowed: true
    warnings: []

  user_summary: ""
```

## 2. 交接给知识资产总控

必须包含：

- 通过质检条目。
- 待验证条目。
- 内部使用条目。
- 拒绝条目。
- AI-Slop 发现。
- 六组分发计划。
- 回流要求。
- 需要总控裁决的问题。

## 3. 交接给六组

每个分发包必须包含：

```yaml
group_delivery_package:
  target_group: ""
  package_name: ""
  task_context: ""
  usable_items: []
  allowed_usage: []
  forbidden_usage: []
  evidence_gaps: []
  feedback_required: []
```

## 4. 老板版摘要

```text
这批知识已经过四个检查：
第一，有没有来源；
第二，能不能公开；
第三，是不是完整的企业、产品、问答或案例；
第四，能不能被六组用到具体经营动作里。

通过的内容会进入正式库；证据不够的内容会进入待验证清单；只能内部使用的内容不会对外公开；泛泛表达和无来源判断会被退回。
```

## 5. 失败输出

```yaml
blocked_output:
  status: "blocked"
  blocked_reason: ""
  blocked_items: []
  return_to_role: ""
  required_fix: []
  allowed_partial_distribution: []
```

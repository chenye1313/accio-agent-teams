# 01 资料接收状态机

版本：v0.1  
适用角色：`01_资料接收与托管师`

## 1. 标准流程

```text
接收资料
  -> 判断来源
  -> 生成编号
  -> 检查可读性
  -> 脱敏与权限标注
  -> 完整度检查
  -> 四库路由建议
  -> 交接给知识资产构建师
```

## 2. 状态机

```yaml
intake_state_machine:
  received:
    next: "source_checked"
    block_if:
      - "material_list empty"
      - "source_owner unknown"

  source_checked:
    next: "asset_indexed"
    block_if:
      - "source_type missing"
      - "received_at missing"

  asset_indexed:
    next: "readability_checked"
    block_if:
      - "asset_id missing"
      - "source_id missing"

  readability_checked:
    next: "privacy_checked"
    block_if:
      - "file unreadable without note"

  privacy_checked:
    next: "permission_checked"
    block_if:
      - "sensitive info not marked"

  permission_checked:
    next: "routing_ready"
    block_if:
      - "permission unknown and intended usage is public"

  routing_ready:
    next: "handoff_ready"
    block_if:
      - "library routing missing"

  handoff_ready:
    next: "asset_builder"
    block_if:
      - "source_index missing"
      - "permission_tags missing"
      - "material_completeness_report missing"
```

## 3. 阻断原则

以下情况必须输出 `blocked`：

- 来源人未知。
- 资料无法编号。
- 敏感信息未标注。
- 公开权限未知但目标用途是对外内容。
- 文件不可读且没有说明。

## 4. 可带缺口通过

以下情况允许 `pass_with_gap`：

- 部分资料缺失，但核心资料可先构建。
- 权限未知资料已被排除在对外范围之外。
- 文件不可读但不影响首轮四库建设。
- 后台数据缺失，但文档和访谈资料足够做初版索引。

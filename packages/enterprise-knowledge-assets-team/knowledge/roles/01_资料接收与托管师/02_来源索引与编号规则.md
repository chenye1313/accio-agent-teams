# 02 来源索引与编号规则

版本：v0.1  
适用角色：`01_资料接收与托管师`

## 1. 编号原则

每份资料必须有：

- `asset_id`
- `source_id`
- `source_type`
- `source_owner`
- `received_at`
- `original_location`
- `privacy_level`
- `permission_status`

## 2. asset_id 命名

```text
KA-{source_type}-{YYYYMMDD}-{sequence}
```

示例：

```yaml
KA-PDF-20260523-001
KA-AUDIO-20260523-002
KA-CHAT-20260523-003
KA-BACKEND-20260523-004
```

## 3. source_type

```yaml
source_type:
  PPT: "公司介绍、产品介绍、课程资料"
  PDF: "公司资料、产品资料、证书、报告"
  DOC: "Word 文档、文字材料"
  SHEET: "Excel、CSV、SKU 表、报价单"
  WEB: "官网、店铺页面、公开网页"
  IMAGE: "图片、截图、海报、后台截图"
  AUDIO: "录音、语音"
  TRANSCRIPT: "录音转写、访谈纪要"
  CHAT: "询盘、WhatsApp、邮件、OKKI、客服聊天"
  BACKEND: "阿里后台、OKKI 数据、成交数据"
  MANUAL: "人工补充、老板回忆、销售复盘"
```

## 4. source_index 模板

```yaml
source_index:
  - source_id: ""
    asset_id: ""
    file_name: ""
    source_type: ""
    source_owner: ""
    received_at: ""
    original_location: ""
    intended_usage: "internal/build/public/training"
    privacy_level: "none/low/medium/high"
    permission_status: "public/anonymized/internal_only/unknown/restricted"
    readability: "pass/failed/unknown"
    notes: ""
```

## 5. 禁止

- 禁止无编号资料进入下游。
- 禁止只记录文件名，不记录来源人。
- 禁止把脱敏文件和原始文件的关系弄断。
- 禁止删除来源线索。

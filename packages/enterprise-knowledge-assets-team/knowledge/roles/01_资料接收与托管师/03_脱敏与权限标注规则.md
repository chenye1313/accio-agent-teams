# 03 脱敏与权限标注规则

版本：v0.1  
适用角色：`01_资料接收与托管师`

## 1. 必须脱敏内容

- 客户姓名。
- 联系方式。
- 邮箱、电话、WhatsApp、微信。
- 未授权客户公司名。
- 具体成交价。
- 合同编号和合同条款。
- 付款信息、账期、收款账户。
- 个人身份信息。

## 2. 可保留内容

- 国家和地区。
- 行业。
- 买家类型。
- 采购场景。
- 金额区间。
- 问题类型。
- 结果类型。
- 产品类型。

## 3. 权限标签

```yaml
permission_status:
  public: "已确认可公开"
  anonymized: "脱敏后可公开或用于课堂演示"
  internal_only: "只能用于内部知识构建"
  unknown: "权限未知，不能对外"
  restricted: "高敏资料，仅总控或授权角色可见"
```

## 4. 脱敏动作记录

```yaml
anonymization_log:
  asset_id: ""
  sensitive_type: "customer_name/contact/price/contract/payment/company_name"
  action: "removed/replaced/generalized/range"
  replacement_rule: ""
  operator_note: ""
```

## 5. 阻断规则

必须阻断：

- `permission_status: unknown` 且 `intended_usage: public`。
- 出现客户联系方式且未处理。
- 具体成交价出现在对外材料。
- 合同或付款信息进入普通资料包。

## 6. 交接要求

交给知识资产构建师时，必须同时交付：

- 原始资料索引。
- 脱敏资料包。
- 脱敏动作记录。
- 权限标签。
- 不可外发清单。

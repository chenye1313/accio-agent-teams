# 05 输出模板与交接格式

版本：v0.1  
适用角色：`01_资料接收与托管师`

## 1. 标准输出

```yaml
intake_custodian_output:
  role_id: "01_资料接收与托管师"
  status: "ready_for_building | pass_with_gap | blocked"

  asset_manifest:
    - asset_id: ""
      file_name: ""
      source_type: ""
      source_owner: ""
      received_at: ""
      readability: "pass | failed | unknown"

  source_index:
    - source_id: ""
      asset_id: ""
      original_location: ""
      source_note: ""

  privacy_gate:
    status: "pass | pass_with_gap | fail"
    sensitive_items: []
    anonymization_required: []
    anonymization_log: []

  permission_tags:
    - asset_id: ""
      permission_status: "public | anonymized | internal_only | unknown | restricted"

  material_completeness_report:
    grade: "A | B | C | D"
    ready_assets: []
    missing_assets: []
    allowed_partial_scope: []

  library_routing_suggestion:
    enterprise_info: []
    product_info: []
    customer_qa: []
    customer_case: []
    evidence_layer: []
    routing_conflicts: []

  handoff_to_asset_builder:
    allowed: true
    package_items: []
    warnings: []

  revision_requests:
    - return_to_role: ""
      issue: ""
      required_fix: ""

  user_summary: ""
```

## 2. 老板版摘要

```text
目前已经收到的资料包括：{ready_assets}
其中需要脱敏或限制使用的是：{privacy_risks}
可以先进入知识构建的资料是：{allowed_partial_scope}
还需要补充的是：{missing_assets}
下一步可以 / 不可以交给知识资产构建师，原因是：{reason}
```

## 3. 交接包

交给知识资产构建师时必须包含：

- `asset_manifest`
- `source_index`
- `anonymized_asset_package`
- `permission_tags`
- `material_completeness_report`
- `library_routing_suggestion`

## 4. 失败输出

```yaml
blocked_output:
  status: "blocked"
  blocked_reason: ""
  return_to_role: "00_Lead知识资产总控 or 用户"
  required_fix: []
  allowed_partial_output: ""
```

# 04 资料完整度与四库路由规则

版本：v0.1  
适用角色：`01_资料接收与托管师`

## 1. 完整度维度

| 维度 | 最低资料 |
|---|---|
| 企业信息 | 公司介绍、官网、PPT、证书、老板访谈 |
| 产品信息 | 产品目录、SKU 表、报价单、详情页、图片视频 |
| 客户问答 | 询盘、聊天、邮件、客服记录、客户录音 |
| 客户案例 | 成交复盘、交付记录、客户反馈、老板或销售回忆 |
| 证据补强 | 阿里后台、OKKI、订单、询盘、产品表现 |

## 2. 完整度等级

```yaml
material_completeness_grade:
  A: "四库都有资料，且至少两类有强证据"
  B: "企业和产品资料完整，问答或案例有部分资料"
  C: "只有公司和产品宣传资料，客户问答和案例不足"
  D: "资料零散，来源不清或不足以进入构建"
```

## 3. 四库路由规则

| 资料类型 | 路由建议 |
|---|---|
| 公司介绍、官网、PPT、资质证书 | 企业信息库 |
| 产品目录、SKU 表、报价单、详情页 | 产品信息库 |
| 询盘、聊天、邮件、客服记录、客户会议 | 客户问答库 |
| 成交复盘、客户反馈、交付记录 | 客户案例库 |
| 老板回忆、销售经验 | 客户案例库线索 / 客户问答库线索 |
| 阿里后台、OKKI、成交数据 | 证据层，补强四库 |

## 5. 禁止

- 禁止把后台数据单独作为对外第五库。
- 禁止把老板经验直接判定为正式案例。
- 禁止把产品目录路由到客户案例库，除非有明确客户使用结果。

# 内容资产生产线与应用路由

## 核心判断

企业知识库基座的四库是底座，不是终点。对阿里外贸企业来说，知识库最终必须能支持经营内容：

```text
品牌品宣
产品表达
结构化商详
FAQ / QAS
询盘跟进
营销推广
社媒 / 独立站
成交复盘
```

因此，总控在调度任务时，必须判断每条知识资产是否能进入内容应用路由。

## 内容资产生产线

```text
企业资料 / 语音沟通 / 老板访谈 / 客户访谈 / 展会交流 / 询盘聊天 / 交易数据
  -> 来源索引与脱敏
  -> 四库候选条目
  -> 内容资产类型识别
  -> 内容应用路由
  -> 应用包生成
  -> 使用反馈回流
```

## 七类内容资产

| 内容资产 | 来源 | 主要用途 |
|---|---|---|
| 企业实力内容 | 企业信息库、证书、工厂资料 | 国际站知识库、公司页、销售背书 |
| 品牌品宣内容 | 老板访谈、公司故事、质量标准 | 对外品牌表达、LinkedIn、独立站 About |
| 产品表达内容 | 产品信息库、SKU、商详、后台数据 | 结构化商详、推广素材、搜索关键词 |
| 买家问题内容 | 客户问答库、询盘、录音、聊天 | FAQ、询盘回复、销售跟进 |
| 信任证据内容 | 客户案例、证书、交易数据 | 商详证明、品宣、案例页 |
| 场景解决方案 | 产品信息、客户案例、买家场景 | 行业方案、选品表达、推广角度 |
| 交易反馈内容 | 询盘、点击、成交、客户反馈 | 优先级排序、内容更新、证据补强 |

## 语音和访谈输入规则

## 内容应用路由

```yaml
content_application_route:
  source_library: "enterprise_info/product_info/customer_qa/customer_case"
  content_layer: "enterprise_strength/brand_promotion/product_expression/buyer_qa/trust_evidence/scenario_solution"
  source_format: "document/voice_transcript/offline_interview/chat_record/backend_data"
  application_target:
    - "alibaba_knowledge_base"
    - "structured_product_detail"
    - "faq_pack"
    - "inquiry_followup"
    - "brand_promotion"
    - "marketing_content"
    - "social_independent_site"
  public_or_internal_boundary: "public/internal_only/needs_validation"
```

## 总控质量门

必须退回：

- 只给转写稿，没有抽取问题、答案、案例、卖点或品宣素材。
- 只做知识条目，没有标注应用场景。
- 老板口述未经证据验证就直接生成对外品宣。
- 客户案例没有公开权限就用于社媒或独立站。
- FAQ 没有 `question / answer / application_scenarios`。
- 产品表达只有参数，没有买家问题、使用场景和证据。

允许通过但标记缺口：

- 品牌故事可先内部使用，但缺公开证据。
- QAS 有真实问题和答案，但缺交易数据验证。
- 产品卖点可用于草稿，但缺图片或证书补强。

## 老板版说明

> 我们不是帮企业写几篇内容，而是把企业每一次沟通、每一份资料、每一个客户问题都变成可重复调用的内容资产。以后商详、FAQ、询盘回复、品宣内容、社媒和独立站，都可以从这套知识库里长出来。

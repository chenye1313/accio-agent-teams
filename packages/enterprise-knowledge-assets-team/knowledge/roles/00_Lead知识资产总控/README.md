# 00 Lead 知识库基座总控私有知识库

## 用途

本目录是 `00_Lead知识库基座总控` 的私有知识库。它不存放原始客户资料，而是存放总控调度企业知识库基座所需的标准、状态机、交接门、质量门、分发规则和输出格式。

总控只读取当前阶段需要的知识切片，不默认全量加载所有企业资料。

## 文件清单

| 文件 | 用途 |
|---|---|
| `01_总控调度状态机.md` | 判断企业知识库基座当前阶段、下一步和阻断条件 |
| `02_四库建设策略与字段门.md` | 定义四库字段、归属规则和方法内沉规则 |
| `03_角色交接门与返工规则.md` | 定义三个执行角色的交接物、最低标准和返工路由 |
| `04_证据隐私质量门.md` | 定义来源、证据等级、脱敏、公开权限和红线 |
| `05_六组分发与回流规则.md` | 定义六组知识分发包和回流协议 |
| `06_输出模板与最终交付格式.md` | 定义总控标准输出 schema 和老板版摘要格式 |

## 使用原则

1. 先判断阶段，再读取对应知识。
2. 先看来源和脱敏，再看内容质量。
3. 四库对外简单，内部保留证据、方法标签和组织实践标签。
4. 任何分发给六组的知识，都必须说明使用场景和回流要求。
5. 红线失败时输出返工角色，不输出假完整结论。

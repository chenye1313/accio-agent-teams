# 03 角色交接门与返工规则

版本：v0.1  
适用角色：`00_Lead知识库基座总控`

## 1. 交接原则

```text
没有通过交接门 = 不允许进入下一阶段。
```

企业知识库基座不是一个大 Agent 一次性完成，而是四个角色依次交接：

```text
知识库基座总控
  -> 资料接收与托管师
  -> 知识库构建师
  -> 知识库质量与分发官
  -> 知识库基座总控放行
```

## 2. 交接检查表

| 阶段 | 上游 | 下游 | 必须有 | 失败则退回 |
|---|---|---|---|---|
| 资料接收 | 总控 | 资料接收与托管师 | 资料清单、来源人、隐私要求、目标四库 | 总控 |
| 资料托管 | 资料接收与托管师 | 知识库构建师 | `source_index`、脱敏资料包、权限标签、完整性报告 | 资料接收与托管师 |
| 知识构建 | 知识库构建师 | 知识库质量与分发官 | 四库候选条目、来源 ID、推断标记、缺失字段 | 知识库构建师 |
| 质量分发 | 知识库质量与分发官 | 总控 | 通过条目、待验证条目、内部条目、六组分发建议 | 知识库质量与分发官 |
| 最终放行 | 总控 | 六组 | 分发包、使用边界、回流要求、不可用清单 | 总控 |

## 4. 最低交付标准

### 资料接收与托管师

```yaml
minimum_output:
  source_index: required
  anonymized_asset_package: required
  permission_tags: required
  material_completeness_report: required
```

### 知识库构建师

```yaml
minimum_output:
  enterprise_info_candidates: required
  product_info_candidates: required
  customer_qa_candidates: required
  customer_case_candidates: required
  source_id_retained: true
  inference_marked: true
```

### 知识库质量与分发官

```yaml
minimum_output:
  approved_items: required
  needs_validation_items: required
  internal_only_items: required
  rejected_items: required
  distribution_plan: required
  feedback_loop_requirements: required
```

## 5. 禁止

- 禁止总控直接替资料接收与托管师编造来源索引。
- 禁止知识库构建师省略来源 ID。
- 禁止知识库质量与分发官放行未脱敏条目。
- 禁止六组收到没有使用场景的知识包。

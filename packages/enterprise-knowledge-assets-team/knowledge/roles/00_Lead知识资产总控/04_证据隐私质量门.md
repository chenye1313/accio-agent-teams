# 04 证据隐私质量门

版本：v0.1  
适用角色：`00_Lead知识库基座总控`

## 1. 来源质量门

每条知识必须有来源。来源类型包括：

```yaml
source_type:
  document: "PPT/PDF/Word/官网/产品目录/证书"
  interview: "老板访谈/销售访谈/团队口述"
  recording: "客户会议录音/语音转写"
  chat: "询盘/WhatsApp/邮件/OKKI/客服记录"
  backend: "阿里后台/OKKI/成交数据/SKU表现"
  manual: "人工补充，必须标明补充人和待验证状态"
```

没有来源的内容只能进入：

```yaml
status: "needs_validation"
conclusion_level: "experience_to_validate"
```

## 2. 证据等级

| 等级 | 定义 | 使用边界 |
|---|---|---|
| A | 后台数据、订单、合同、成交记录、客户反馈、认证证书 | 可作为确定结论 |
| B | 文档、聊天、录音、脱敏案例、交付记录 | 可作为结论，但建议标注来源 |
| C | 老板或销售口述，有业务合理性但缺少外部验证 | 只能作为待验证经验 |
| D | 无明确来源、漂亮口号、泛泛判断 | 不可进入正式库 |

## 3. 隐私质量门

以下内容必须脱敏：

- 客户姓名。
- 联系方式。
- 邮箱、WhatsApp、电话。
- 未授权公司名。
- 具体成交价。
- 合同、付款、账期信息。
- 内部员工个人隐私。

可保留：

- 国家。
- 行业。
- 买家类型。
- 采购场景。
- 金额区间。
- 问题类型。
- 结果类型。

## 4. 公开权限

```yaml
public_permission:
  public: "已确认可公开"
  anonymized: "可脱敏后公开"
  internal_only: "只能内部使用"
  unknown: "权限未知，禁止对外使用"
```

## 5. 质量红线

必须拒绝：

- 来源缺失。
- 隐私未处理。
- 证据等级缺失。
- 企业优势无证据。
- 产品卖点无使用场景。
- 问答无买家语境。
- 案例无问题、方案或结果。
- 方法经验伪装成事实。
- 公开权限未知但被用于对外内容。

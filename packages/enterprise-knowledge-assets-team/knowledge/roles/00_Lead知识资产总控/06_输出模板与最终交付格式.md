## 2. 老板版摘要

老板版只讲业务话：

```text
现在我们先做企业知识库基座：企业知识库基座。
这一步不是直接让 AI 写内容，而是先把企业资料、产品资料、客户问题和真实案例整理成四个库。
目前可以进入下一步的是：{what_is_ready}
还需要补的是：{what_needs_review}
下一步交给：{next_role_or_task}
这一步完成后，后面的经营、选品、内容、运营、成交、私域六组才有企业自己的知识可以调用。
```

## 3. 内部版摘要

内部版保留证据、状态和返工：

```yaml
internal_summary:
  stage_status: ""
  gate_status: ""
  evidence_gaps: []
  privacy_risks: []
  return_to_role: ""
  allowed_next_step: ""
```

## 4. 失败输出

红线失败时，不允许输出“已完成”口径，只能输出：

```yaml
blocked_output:
  status: "blocked"
  blocked_reason: ""
  return_to_role: ""
  required_fix: ""
  allowed_partial_output: ""
```

## 5. 禁止

- 禁止客户版出现大量状态码。
- 禁止把内部证据等级当成对外表达。
- 禁止在缺少来源和脱敏时说“可以进入分发”。
- 禁止把 `pass_with_gap` 讲成完全通过。

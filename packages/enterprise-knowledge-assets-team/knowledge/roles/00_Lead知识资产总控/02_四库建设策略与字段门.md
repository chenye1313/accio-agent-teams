# 02 四库建设策略与字段门

版本：v0.1  
适用角色：`00_Lead知识库基座总控`

## 1. 四库外显原则

对老板和业务团队只讲四库：

1. 企业信息库。
2. 产品信息库。
3. 客户问答库。
4. 客户案例库。

方法经验不外显成第五库，只作为内部标签沉淀在：

- 客户案例库：`reusable_method`
- 客户问答库：`answer_structure / application_scenarios / recommended_usage / tags`
- 组织实践标签：`scenario / process / role / judgment / action / standard / mechanism / feedback`

## 2. 主库归属规则

| 知识类型 | 主库 | 可关联库 |
|---|---|---|
| 公司介绍、工厂能力、认证、团队 | 企业信息库 | 客户案例库 |
| 产品线、SKU、参数、材质、MOQ、交期 | 产品信息库 | 客户问答库 |
| 客户常问问题、异议、报价、交期、售后 | 客户问答库 | 产品信息库、客户案例库 |
| 成交过程、客户问题、解决方案、结果 | 客户案例库 | 企业信息库、产品信息库、客户问答库 |
| 老板经验、销售技巧、谈判判断 | 不单列主库 | 案例 `reusable_method` 或问答 `recommended_usage` |
| 阿里后台、OKKI、询盘、成交数据 | 证据层 | 补强四库和六组判断 |

## 3. 字段最低门槛

### 企业信息库

```yaml
required_fields:
  - company_profile
  - capabilities
  - service_scope
  - advantages
  - evidence
quality_gate:
  - advantages_must_have_evidence
  - certifications_must_have_source
  - service_scope_must_be_specific
```

### 产品信息库

```yaml
required_fields:
  - product_line
  - sku
  - features
  - use_cases
  - customization
  - delivery
  - proof
quality_gate:
  - features_must_map_to_use_case
  - delivery_terms_must_mark_source
  - proof_must_not_be_empty_for_main_product
```

### 客户问答库

```yaml
required_fields:
  - question
  - buyer_context
  - answer
  - answer_structure
  - application_scenarios
  - objection_type
  - tags
  - question_keywords
  - source
  - recommended_usage
quality_gate:
  - question_must_preserve_original_or_normalized_form
  - buyer_context_must_not_be_empty
  - answer_must_not_be_generic
  - answer_must_have_core_point_or_action_step
  - application_scenarios_must_not_be_empty
  - tags_and_keywords_must_support_retrieval
```

### 客户案例库

```yaml
required_fields:
  - buyer_type
  - problem
  - conversation_summary
  - solution
  - result
  - reusable_method
  - evidence_level
quality_gate:
  - case_must_include_problem_solution_result
  - public_permission_must_be_set
  - reusable_method_must_be_marked_internal_if_unverified
```

## 4. 四库冲突处理

```yaml
library_conflict_rule:
  if_product_fact_appears_in_case:
    primary_library: "customer_case"
    cross_reference: "product_info"
  if_customer_question_mentions_sku:
    primary_library: "customer_qa"
    cross_reference: "product_info"
  if_company_advantage_only_from_case:
    primary_library: "customer_case"
    cross_reference: "enterprise_info"
    status: "supported_hypothesis"
  if_method_from_boss_memory:
    primary_library: "none"
    target_field: "reusable_method or recommended_usage"
    status: "experience_to_validate"
```

## 5. 禁止

- 禁止把无证据优势放进企业信息库正式条目。
- 禁止把产品参数和产品卖点混为一谈。
- 禁止问答没有买家语境。
- 禁止问答没有应用场景、标签和问题关键词。
- 禁止案例没有结果。
- 禁止方法经验外显成第五库。
